Version: 2.0.0
Date: 2026-06-30

  Features:

    - Updated the mod for Factorio 2.1.
    - Added Space Age cargo bay unloading distance infinite research for the Factorio 2.1 landing pad unloading bay. Each level adds 10 tiles by default, requires the vanilla unloading bay unlock, uses all available science packs, and has configurable enable, cost, growth, max-level, and research-time startup settings.
    - Added Space Age cargo landing pad count infinite research as a disabled-by-default sandbox option. Each level adds one additional cargo landing pad per surface by default, uses all available science packs, and starts at a deliberately expensive 1,000,000 research units with 10x growth.
    - Added local fixture mods and a PowerShell validation runner for static checks and optional Factorio runtime load checks.
    - Added a startup-controlled generation diagnostics report for generated and skipped streams and base extensions.
    - Added a startup-controlled recipe match diagnostics report for troubleshooting which recipes each productivity stream captured.
    - Added duplicate recipe match diagnostics when generation or recipe-match diagnostics are enabled.
    - Added a lab incompatibility policy setting so users can choose between reducing incompatible science-pack sets to a lab-compatible subset or skipping the affected technology.
    - Reworked generated research compatibility for Factorio 2.1 by discovering science packs from active lab inputs instead of assuming science packs are tool prototypes.
    - Expanded science-pack productivity from active lab inputs so custom item-based science packs can receive productivity effects when their recipes are visible.
    - Moved generated technology creation to data-final-fixes so recipes, labs, items, science packs, and technologies from other mods are visible before scanning.
    - Added Wube-style layered technology constant icon overlays for generated research technologies.
    - Added lab-compatible science-pack validation so generated technologies are skipped or reduced instead of becoming unresearchable.
    - Added support for Factorio 2.1 recipe categories and stream match filters.
    - Added a startup setting to control whether vanilla weapon shooting speed technologies are adjusted.
    - Changed the late-game science gate setting to be disabled by default, so generated technologies are not hidden until space science or promethium science unless users opt in.
    - Added a science-pack ingredient policy setting so users can keep configured packs, add the end-game science pack to every generated technology, or require every active lab science pack including compatible modded packs.
    - Fixed Space Age cargo logistics research so cargo landing pad count and cargo bay unloading distance are skipped when Space Age is disabled.
    - Added in-game descriptions that explain the recipe productivity cap and lab-compatible science-pack behavior.

  Generator changes:

    - Generated stream prerequisites now include declared required technologies, so prototype availability checks and actual technology progression stay aligned.
    - Direct-effect streams can now provide a specific localized description instead of always using the generic direct-effect description.
    - Optional DLC-shaped direct-effect streams now declare explicit prototype requirements instead of hard-requiring Space Age.
    - Added optional load ordering for Elevated Rails, Recycler, Quality, and Space Age so any installed official DLC combination is visible before MIR generation.
    - Removed hard Space Age generation gates from MIR streams; unavailable targets now skip through recipe, item, technology, or ammo-category availability checks.
    - Science-pack productivity now reuses Space Age's research productivity technology art when available while keeping the generated productivity corner badge.
    - Science-pack productivity can use science pack item icons through item fallback lookup.
    - Recipe-productivity streams can now opt into dynamic target items from active lab inputs.
    - Recipe matching now skips hidden recipes and recycling recipes by default unless a stream opts in.
    - Narrowed breeding productivity matching so biochamber category membership alone no longer pulls every biochamber recipe into breeding productivity.
    - Generated recipe effect order is now deterministic.
    - Removed unused base-extension growth inference helpers and kept generation diagnostics behind the debug report setting.
    - Reworked base-tech extension science packs to inherit from the last vanilla level by default and then add only the intended extra packs.
    - Added science-pack prerequisite repair for generated base-tech extensions so added packs also add their unlock prerequisites.
    - Added detection for equivalent existing infinite base-tech extensions so duplicate infinite continuations are skipped instead of generated again.
    - Added logic to keep another mod's competing infinite continuation when the preference setting is disabled.
    - Fixed base-technology extension settings so entering 0 for base cost, growth factor, or research time now derives that value from the vanilla technology chain instead of falling back to this mod's configured defaults.
    - Base-technology extensions now skip generation when a positive max-level setting is below the first generated continuation level.
    - Kept declared required-technology prerequisite handling centralized in the shared prerequisite builder.
    - Split the prototype utility facade into focused modules for lookup, science-pack validation, recipe matching, icon construction, and deepcopy.
    - Split stream definitions into productivity and direct-effect domain modules.
    - Added compatibility profile scaffolding for future mod-specific stream patches.
    - Compatibility profile patches now support append-style fields for extending stream item, pattern, exclusion, and group arrays without replacing the base stream data.

  Compatibility:

    - Raised the base dependency to Factorio 2.1.8 or newer because cargo bay unloading distance uses the 2.1.8 technology modifier API.
    - Added optional official DLC ordering floors of 2.1.8 or newer for Elevated Rails, Recycler, Quality, and Space Age.
    - Kept compatibility-mod dependencies out of release metadata so the mod page stays clean; known integrations are handled opportunistically from visible prototypes and safe skip checks.
    - Added generic competing base-extension cleanup and used it for Better Robots Extended worker robot storage infinite extensions.
    - Competing base-extension cleanup now runs only when this mod's matching base extension is enabled.
    - Moved known competing recipe-productivity cleanup after MIR stream generation and made it inspect actual generated MIR recipe-productivity effects before removing another mod's competing technology.
    - Narrowed known competing recipe-productivity cleanup to infinite technologies only so finite upgrade chains from other mods are not removed by the generic cleanup path.
    - Compatibility cleanup now removes dangling prerequisite references when deleting known competing technologies.
    - Changed worker robot storage extension science packs toward utility and electromagnetic science in Space Age contexts.
    - Added opportunistic compatibility coverage for Advanced Solar HR, OCs Ammo and Armor, OCs Stone Casting, Fluid Quality Imprinting, and Plates n Circuit Productivity without declaring those mods as dependencies.
    - Added explicit bio-explosives matching for OCs Ammo and Armor's biochamber explosives recipes.
    - Made Space Age turbo belt, underground belt, and splitter productivity coverage explicit at the 1% tier.
    - Changed rocket shooting speed research to use the base-game rocketry technology icon instead of a Space Age rocket turret icon.
    - Split cannon-shell shooting speed into its own dedicated cannon shooting speed research instead of bundling it into rocket shooting speed.
    - Changed the default vanilla weapon shooting speed adjustment mode to `off`; users can opt into dedicated-tech or always-on cleanup from startup settings.
    - Removed selected competing productivity technologies when this mod is configured to prefer its own generated versions.
    - Better supports Space Age optional installs and Space Age without Quality.
    - Better supports mods that add science packs as ordinary items or add lab inputs before this mod's final-fixes phase.
    - Better supports Castra, PlanetLib-style planet mods, and other custom science-pack mods through lab-input discovery, lab-compatible ingredient validation, and dynamic science-pack productivity targets.
    - Known limitation: mods that mutate recipes or labs later in data-final-fixes still require load-order compatibility.

  Localization:

    - Replaced placeholder localization text in German, Spanish, French, Italian, Polish, Brazilian Portuguese, and Simplified Chinese.
    - Updated Russian localization coverage and terminology to match the current English locale keys, including community-provided improvements from the unreleased pre-2.0 history.
    - Updated in-game mod descriptions to cover productivity research, direct-effect bonus research, and Space Age cargo logistics research.
    - Split generated-stream research-time setting text from base-technology extension research-time text so the fallback behavior is described accurately.
    - Clarified base-technology extension research-time setting text so only an explicit 0 promises vanilla research-time reuse.
    - Moved Spanish localization to Factorio's supported es-ES locale code.
    - Removed empty placeholder locale folders so only complete locale packs are shipped.
    - Moved heavy ammunition, armor component, and stone product research names from inline English strings into synchronized locale keys.
    - Replaced leftover English cannon shooting speed text in translated locale files.
    - Added locale validation tooling and maintenance notes to keep translated files synchronized with the English fallback.
    - Added validation checks for the no-compatibility-mod-dependencies release policy, locale synchronization, and Factorio runtime exit codes.
    - Added validation for Factorio's strict changelog section format.
    - Strengthened validation so versioned compatibility-mod dependencies are rejected, the release zip is checked for the expected root and metadata, and the runtime log path can be overridden.
    - Strengthened runtime fixture validation so the custom item-based science pack must be present in the generated science-pack productivity diagnostics.
    - Added a post-MIR runtime fixture assertion that fails loading if the custom item-based science-pack recipe does not receive a science-pack productivity effect.
    - Strengthened runtime validation so a missing expected Factorio log is a failure instead of silently skipping log assertions.
    - Strengthened release package validation so core data-stage files, stream modules, and shared recipe/science-pack helpers must be present in the zip.
    - Strengthened release package validation so key packaged source files must match the repository copy, catching stale archives with otherwise valid metadata.
    - Strengthened release package validation so README, changelog, documentation, and locale files must match the repository copy.
    - Added runtime fixture validation for the `skip` lab incompatibility policy.

  Documentation:

    - Overhauled the README with a full user and developer technical specification, including research catalogs, settings, defaults, compatibility behavior, validation commands, and troubleshooting notes.
    - Added architecture documentation for the data-stage pipeline, utility modules, stream config domains, compatibility profiles, diagnostics, and validation harness.
    - Added GitHub Actions static validation for pushes, pull requests, and manual runs.
    - Added a manual validation matrix item for the disabled-by-default cargo landing pad count research when it is forced enabled.
    - Documented the dependency-free compatibility policy and package validation requirements.
    - Rebuilt the changelog with historical entries back to the original 1.0.0 release.

  Migration:

    - No existing prototype IDs were renamed. No migration is required.

Version: 1.0.0
Date: 2025-10-25

  Features:

    - Initial public release for Factorio 2.0 with optional Space Age support.
    - Added generated infinite recipe productivity technologies for many intermediate items and resources not covered by vanilla productivity researches.
    - Added per-technology startup enable switches for every generated research stream.
    - Added automatic recipe scanning: the mod finds recipes that output the covered item or item family and creates `change-recipe-productivity` effects for those recipes.
    - Added conflict trimming so recipes already covered by existing productivity technologies are skipped.
    - Added Space Age-aware science-pack selection and prerequisites for generated technologies.
    - Added a default infinite research cost curve of `8000 * 2^(L-1)` with 60 seconds per research unit.

  Initial research coverage:

    - Added productivity streams for breeding, plastic, sulfur, batteries, explosives, iron gear wheels, iron sticks, copper cable, electronic circuits, advanced circuits, processing units, low density structures, rocket fuel, copper plates, iron plates, engine units, electric engine units, and flying robot frames.
    - Added Space Age and overhaul-oriented productivity streams for tungsten, holmium, supercapacitors, superconductors, bioflux, carbon fiber, lithium, and quantum processors.
    - Added grouped productivity streams for modules, belts, inserters, bullets, and rockets, with lower per-level gains for higher-tier items inside those groups.

  Notes:

    - The technologies are intentionally infinite, but Factorio's recipe productivity cap still applies.
    - The mod was created to make megabasing easier on lower-end hardware while still preserving Factorio's normal production and logistics constraints.
