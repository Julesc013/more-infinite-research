Version: 2.0.0
Date: 2026-06-30

  Features:

    - Updated the mod for Factorio 2.1.
    - Added Space Age cargo bay unloading distance infinite research for the Factorio 2.1 landing pad unloading bay. Each level adds 10 tiles by default, requires the vanilla unloading bay unlock, uses all available science packs, and has configurable enable, cost, growth, max-level, and research-time startup settings.
    - Added local fixture mods and a PowerShell validation runner for static checks and optional Factorio runtime load checks.
    - Added a startup-controlled generation diagnostics report for generated and skipped streams and base extensions.
    - Reworked generated research compatibility for Factorio 2.1 by discovering science packs from active lab inputs instead of assuming science packs are tool prototypes.
    - Moved generated technology creation to data-final-fixes so recipes, labs, items, science packs, and technologies from other mods are visible before scanning.
    - Added Wube-style layered technology constant icon overlays for generated research technologies.
    - Added lab-compatible science-pack validation so generated technologies are skipped or reduced instead of becoming unresearchable.
    - Added support for Factorio 2.1 recipe categories and stream match filters.
    - Added a startup setting to control whether vanilla weapon shooting speed technologies are adjusted.
    - Added in-game descriptions that explain the recipe productivity cap and lab-compatible science-pack behavior.

  Generator changes:

    - Generated stream prerequisites now include declared required technologies, so prototype availability checks and actual technology progression stay aligned.
    - Direct-effect streams can now provide a specific localized description instead of always using the generic direct-effect description.
    - Space Age-only direct-effect streams now declare explicit optional prototype requirements.
    - Science-pack productivity can use science pack item icons through item fallback lookup.
    - Recipe matching now skips hidden recipes and recycling recipes by default unless a stream opts in.
    - Generated recipe effect order is now deterministic.
    - Split the prototype utility facade into focused modules for lookup, science-pack validation, recipe matching, icon construction, and deepcopy.
    - Split stream definitions into productivity and direct-effect domain modules.
    - Added compatibility profile scaffolding for future mod-specific stream patches.

  Compatibility:

    - Raised the base dependency to Factorio 2.1.8 or newer because cargo bay unloading distance uses the 2.1.8 technology modifier API.
    - Added optional load-order dependencies for Advanced Solar HR, OCs Ammo and Armor, OCs Stone Casting, Fluid Quality Imprinting, and Plates n Circuit Productivity so their late recipe or technology changes are visible before this mod generates research.
    - Added explicit bio-explosives matching for OCs Ammo and Armor's biochamber explosives recipes.
    - Made Space Age turbo belt, underground belt, and splitter productivity coverage explicit at the 1% tier.
    - Removed selected competing productivity technologies when this mod is configured to prefer its own generated versions.
    - Better supports Space Age optional installs and Space Age without Quality.
    - Better supports mods that add science packs as ordinary items or add lab inputs before this mod's final-fixes phase.
    - Known limitation: mods that mutate recipes or labs later in data-final-fixes still require load-order compatibility.

  Localization:

    - Replaced placeholder localization text in German, Spanish, French, Italian, Polish, Brazilian Portuguese, and Simplified Chinese.
    - Updated Russian localization coverage and terminology to match the current English locale keys.
    - Moved Spanish localization to Factorio's supported es-ES locale code.
    - Removed empty placeholder locale folders so only complete locale packs are shipped.
    - Added locale validation tooling and maintenance notes to keep translated files synchronized with the English fallback.

  Documentation:

    - Added architecture documentation for the data-stage pipeline, utility modules, stream config domains, compatibility profiles, diagnostics, and validation harness.
    - Rebuilt the changelog with historical entries back to the original 1.0.0 release.

  Migration:

    - No existing prototype IDs were renamed. No migration is required.

Version: 1.0.0
Date: 2025-10-25

  Features:

    - Initial public release for Factorio 2.0 with optional Space Age support.
    - Added generated infinite recipe productivity technologies for many intermediate items and resources not covered by vanilla productivity researches.
    - Added per-technology startup enable switches for every generated research stream.
    - Added automatic recipe scanning: the mod finds recipes that output the covered item or item family and creates `change-recipe-productivity` effects for those recipes.
    - Added conflict trimming so recipes already covered by existing productivity technologies are skipped.
    - Added Space Age-aware science-pack selection and prerequisites for generated technologies.
    - Added a default infinite research cost curve of `8000 * 2^(L-1)` with 60 seconds per research unit.

  Initial research coverage:

    - Added productivity streams for breeding, plastic, sulfur, batteries, explosives, iron gear wheels, iron sticks, copper cable, electronic circuits, advanced circuits, processing units, low density structures, rocket fuel, copper plates, iron plates, engine units, electric engine units, and flying robot frames.
    - Added Space Age and overhaul-oriented productivity streams for tungsten, holmium, supercapacitors, superconductors, bioflux, carbon fiber, lithium, and quantum processors.
    - Added grouped productivity streams for modules, belts, inserters, bullets, and rockets, with lower per-level gains for higher-tier items inside those groups.

  Notes:

    - The technologies are intentionally infinite, but Factorio's recipe productivity cap still applies.
    - The mod was created to make megabasing easier on lower-end hardware while still preserving Factorio's normal production and logistics constraints.
