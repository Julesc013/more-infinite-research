# 3.0.0 Local Library Gate Summary

Updated: 2026-07-07

This release artifact records the local mod-library catalog gates used during the MIR 3 transition. The JSON outputs are generated under ignored `build/` paths and are not release-zip contents.

## Commands

```powershell
.\scripts\Test-MIRLocalModLibraryCatalog.ps1 `
  -LocalModLibraryDirs "C:\Projects\Factorio\testmods_2.1" `
  -ScenarioPath "fixtures\compat-matrix\local-library-scenarios.json" `
  -MinimumZipCount 1 `
  -OutputPath "build\local-catalog-2.1.json"

.\scripts\Test-MIRLocalModLibraryCatalog.ps1 `
  -LocalModLibraryDirs "C:\Projects\Factorio\testmods_2.0" `
  -ScenarioPath "fixtures\compat-matrix\local-library-scenarios-2.0.json" `
  -MinimumZipCount 1 `
  -OutputPath "build\local-catalog-2.0.json" `
  -AllowMissingScenarioMods
```

| `2.0` | `2026-07-07T17:58:04.0591785+10:00` | 324 | 314 | 11 | 19 | 10 | 9 | Pass with recorded missing roots |

The `2.1` catalog gate is complete for the committed local-library scenarios: all 55 scenario root mods were present in `C:\Projects\Factorio\testmods_2.1`.

The `2.0` catalog gate was run with `-AllowMissingScenarioMods` so the current legacy-library coverage gap is explicit and non-destructive. The missing root mods were:

```text
angelsindustries

bobplates
bobtech

```

This is local library completeness debt for the future `1.9.3` or other legacy backport lane. It is not evidence that the MIR 3 source tree changed generated technologies incorrectly.
