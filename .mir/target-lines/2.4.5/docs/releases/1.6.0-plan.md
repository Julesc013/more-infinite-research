# MIR 1.6.0 Factorio 0.16 Plan

## Scope

MIR 1.6.0 is the first current Factorio 0.16 plan. It is a reduced old-science native-infinite edition built only from target-proven portable contracts. Archived 0.16 planning records with 1.8.x version slots remain historical and do not authorize implementation.

## Non-goals

- Do not simulate recipe-productivity technology effects.
- Do not import Factorio 2.x metadata, DLC, runtime storage, settings, icons, or prototype fields.
- Do not claim parity with MIR 3.x or 2.x.

## Required Changes

- Build an exact Factorio 0.16 target profile with a positive native-effect allowlist.
- Map semantic science roles to the target-era science pack prototypes and lab inputs.
- Adapt technology, recipe, icon, locale, and runtime-state shapes to Factorio 0.16.
- Preserve stable portable plan, owner, prerequisite, and deterministic-ID contracts where the target supports them.
- Emit only target-proven native infinite technologies; fail closed for every unsupported action.

## Validation

- Run static package and architecture checks on `tmp/0.16`.
- Load the exact candidate in the matching Factorio 0.16 binary.
- Prove every emitted effect, count formula, prerequisite, science ingredient, icon, and locale key.
- Create, reload, and upgrade a representative save when a prior target release exists.
- Inspect the release ZIP for modern metadata or excluded development files.

## Compatibility Impact

This is a new reduced target line. It must not change modern generated IDs, settings, behavior, or frozen archives.

## Release Notes

Describe 1.6.0 as a reduced Factorio 0.16 native-infinite edition with an explicit target-proven feature list.

## Rollback

Discard the unpublished candidate and return to the branch source anchor. Never replace a published archive.
