## Change Rule

3.1 may replace internal authorities and implementation paths, but parity must be explained at these public boundaries. A changed hash is not automatically a regression; it is a review trigger that needs semantic fixtures, migration disposition where applicable, and an updated characterization delta.

## Deterministic Package Delta

Commit `9a04ded` introduced sorted fixed-metadata ZIP construction. The first two-worktree proof correctly failed because one checkout exposed CRLF text and the detached checkout exposed LF. Commit `5412d8dd7b65f9b90e5bd64acbbd74b04aa3f750` remediated the drift by writing all package text as UTF-8 without BOM and canonical LF endings.

Two clean worktrees at the repaired commit produced byte-identical 342043-byte archives with 136 sorted file entries and SHA-256 `B1EABCAFF4F802A104AF4D31CE7CE98A98DD0A59A3F5D7F60C97A2015A5FB81A`. `.mir/evidence/3.1.0-deterministic-package-proof.json` records the failure, repair, and final proof. This is a development-gate identity, not the final 3.1 release-candidate hash.
