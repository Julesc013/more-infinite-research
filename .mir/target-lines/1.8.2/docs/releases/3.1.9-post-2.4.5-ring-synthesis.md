Released MIR 2.4.5 remains anchored by tag and legacy commit `7e4b6c530cfcc5b2e1429c4e9f4ccb0d6d3b42a4`, package-source commit `47bcc80c8a70b41dbbb4f8d80b2b9433d0a10d3f`, and archive SHA-256 `7649824B72247AA38F05661422DFDEE7C729B21CC73A0A35D2455443B45D39F8`.

The only tree change after the 3.1.9 tag is the addition of the immutable 2.4.0 and 2.4.5 distribution archives; the 3.1.9 package source and released bytes are unchanged.

The complete 2.4.0-to-2.4.5 first-parent delta is classified in `.mir/evidence/ring-1.1-1.0/legacy-2.4.5-portability-audit.json`. Disabled-science prerequisite handling, paired native-owner cost semantics, progress retention, checkpoint and resume behavior, first-failure preservation, duplicate static suppression, strict ecosystem suppression, build-once qualification, and exact sealing are already present on the modern line. The generic backport source-lock validator is the only newly returned harness capability.

Exact Factorio 1.1.110 and 1.0.0 package probes subsequently exposed three additional portable boundaries. `mod-data` is now an explicit positive target-profile capability, so the emitter does not create an unsupported prototype family on 1.x targets. Source-lock map reads preserve empty and singleton JSON objects as arrays, and reduced legacy validation may retain off-target modern fixtures while target scenario declarations select explicitly retargeted fixtures. Because the `mod-data` correction changes canonical package source, the development anchor must be requalified before either target can be sealed.

Factorio 2.0 metadata, dependency floors, prototype shapes, runtime migration snapshots, reduced defaults, target fixtures, candidate evidence, release wording, and distribution bytes remain excluded from modern source. The released `legacy` history is not merged upward.

The branch `DEV_RING_1_1_1_0_ANCHOR` freezes the canonical source for the Factorio 1.1 and Factorio 1.0 reconstructions. Each target source lock must resolve that branch to one full commit, name the released 3.1.9 and 2.4.5 anchors, classify every canonical feature, declare every package-source adaptation, and bind the exact candidate seal.

The next Factorio 0.17 and 0.16 ring is documentation-only during this run. It may begin only after both target publications are verified and a separate execution authority is issued.
