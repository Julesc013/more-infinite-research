# 2.4.0 Roadmap

MIR 2.4.0 is the Factorio 2.0 companion to the released 3.1.5 compiler campaign. It is not a metadata-only downgrade and does not promise Factorio 2.1 feature parity.

## Source And Sequencing

1. Preserve the exact internal `2.5.0` candidate commit and digest as superseded safety evidence.
2. Use released `3.1.5` as the final canonical source anchor and fold its portable compiler work into `tmp/2.0`.
3. Collapse the release identity to `2.4.0`; remove release-facing `2.5.0` metadata, fixtures, and archive bytes.
4. Apply only Factorio 2.0 metadata, effect, dependency, and API cuts declared by the positive target-capability model.
5. Qualify and seal the exact archive, promote the same candidate to `legacy`, then stop for maintainer tag and publication.

## Required 2.4.0 Gates

- Every shipped stream, setting, effect, pipeline pass, and runtime handler has an explicit Factorio 2.0 support classification.
- Final compatibility profile output cannot recreate a target-filtered stream.
- Duplicate stream IDs fail before prototype emission.
- The same transactional ownership, scenario-completeness, clean-harness, and prerequisite-rewire fixtures pass on a real Factorio 2.0 binary.
- Base and Space Age/Quality load checks, exact-dist checks, selected local mod library gates, existing 2.3.x save checks, and settings retention pass.
- Public docs describe 2.4.0 as the maintained Factorio 2.0 port, not as an identical 3.1.0 build.

## Current Execution State

The source backport, release-identity collapse, native-owner implementation, qualification, sealing, `legacy` promotion, and publication are complete. The published release commit is `584b398f98d3e317fac31cba63edcb11360a5bb1`, and the immutable archive SHA-256 is `4BA19EA071E6359BC25C58CCD8F65CAF81B4AA675496E2F53175A996C791470C`.

## Beyond 2.4.0

MIR 2.4.5 is the active Factorio 2.0 startup-safety hotfix. Older target work remains outside this hotfix run until 2.4.5 reaches its maintainer release boundary.
