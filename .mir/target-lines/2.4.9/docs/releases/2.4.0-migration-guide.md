# MIR 2.4.0 Migration Guide

MIR 2.4.0 is the maintained Factorio 2.0 companion to MIR 3.1.5. Install it only on Factorio 2.0. Factorio 2.1 users should use the MIR 3.x line.

## Upgrading From 2.3.5

Back up the save, replace the 2.3.5 archive with the 2.4.0 archive, keep the existing startup settings, and load the save normally. Do not keep both MIR archives enabled in the same mods directory.

Generated technology IDs, setting IDs, profile fields, runtime namespaces, and the supported Factorio 2.0 behavior surface remain compatible by contract. The release gate must prove the exact 2.3.5-to-2.4.0 archive transition retains non-default settings, research level, persisted fixture state, and applied runtime effects before publication.

## Intentional Target Difference

Factorio 2.0 does not receive the `max-cargo-bay-unloading-distance` or `cargo-landing-pad-count` technology modifiers introduced on the Factorio 2.1 line. Their streams and settings are absent rather than simulated. Other features remain governed by the positive Factorio 2.0 target profile and concrete prototype availability.

## Troubleshooting

If a generated technology is missing, enable MIR generation diagnostics and inspect `factorio-current.log` for the stream key and reason code. A missing required prototype, unavailable science path, existing infinite owner, loop-risk decision, or target requirement is an intentional skip unless the log reports an error.
