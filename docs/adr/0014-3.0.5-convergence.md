# ADR 0014: MIR 3.0.5 Convergence

## Context

The Factorio 2.0, 1.1, 0.18, 1.0, and 0.17 ports produced portable fixes, fixtures, and clearer target boundaries. Directly merging their implementations would also import target-local metadata, disabled modern features, duplicated conditionals, and older runtime assumptions.

The Factorio 0.17 release exposed a general science prerequisite inference bug: an unlock effect on a disabled tutorial technology was treated as a valid freeplay progression gate. The correct repair belongs in the shared science capability and generated graph invariant, not in a `basic-mining` exception.

## Decision

MIR 3.0.5 is a bounded convergence and compatibility-hardening release:

```text
behavioral superset of intentionally supported MIR 3 behavior
implementation subset of the combined historical branches
```

`dev` remains the canonical implementation. Historical branches provide behavioral evidence, release provenance, fixtures, and migration requirements. Each accepted behavior is classified in `.mir/convergence.yml` and implemented through the existing MIR module boundaries.

The bounded refactor covers:

- generated science prerequisite and graph safety;
- canonical target capability profiles;
- explicit target-selected runtime state;
- validation scenario grouping and structured partial summaries;
- separate settings codec and reduced-surface fixtures;
- coverage-aware weapon speed ownership;
- release and planning documentation reconciliation.

## Non-Goals

3.0.5 does not add a general plugin framework, new overhaul support, new productivity families, Factorio 0.16 science identifiers, broad module renames, or a replacement compiler. Factorio already provides the external mod boundary; any new public MIR extension API requires separate 3.1 design and conformance evidence.

## Consequences

Target knowledge becomes explicit and testable rather than duplicated in Lua and PowerShell. Runtime persistence uses the backend declared for the selected target. Generated technology graphs fail validation when prerequisites are disabled, cyclic, missing, or structurally unreachable. Validation can preserve useful partial evidence when a long run is interrupted.

The patch remains compatible with the public 3.0.x contract. Existing saved startup settings are not migration-forced. A changed weapon-overlap default applies only when Factorio creates a new configuration without a stored value.

## Reversal

Each convergence capability is committed independently. The baseline tag `pre-3.0.5-synthesis` identifies the modern implementation before convergence. An individual behavior can be reverted without restoring target-local branch code or changing generated technology IDs.
