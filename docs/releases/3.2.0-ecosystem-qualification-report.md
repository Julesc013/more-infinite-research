# 3.2.0 Ecosystem Qualification Report

## K2 And Space Ecosystems

Krastorio 2 base passed with seven resolved third-party mods. The current Factorio 2.1 Space Age path, `Krastorio2-spaced-out` plus its enhancement root, also passed with seven resolved third-party archives, the four official Space Age archives, zero dependency failures, and zero MIR audit rows.

The separate legacy `Krastorio2` plus `space-exploration` scenario did not reach a valid mod load. Its locally available 19-archive closure lacks `space-exploration-graphics-2`, `space-exploration-menu-simulations`, and `space-exploration-postprocess`. Factorio rejected Space Exploration for those missing required dependencies. That is an input-closure blocker, not evidence of a MIR runtime defect and not a load claim.

## PY Boundary

The local 602-archive library contains `py_productivity` but not the governed `pymodpack` root. Offline resolution therefore recorded one dependency failure and no executed PY closure. The Factorio process could load MIR without the requested root, but the campaign correctly marked the scenario failed because the requested dependency closure was absent. No PY compatibility claim follows.

The execution environment exposed no Mod Portal download token, so the missing archives could not be acquired through the governed resolver without new credentials or a supplied exact local closure.

## Claim Boundary

A passing campaign row proves only that its exact archive closure loaded with MIR under the recorded Factorio binary. It does not qualify each ecosystem independently for semantic cooperation, family support, save migration, or future archive versions. A blocked row proves only that the required input closure was unavailable.

The machine-readable aggregate is `.mir/evidence/3.2.0-ecosystem-qualification-summary.json`.
