## Root Cause

The defect was not normal K2SO behavior. C14 loaded MIR before Krastorio 2 and Krastorio 2 Spaced Out final data updates. MIR therefore compiled against K2's temporary early research graph, where `logistic-science-pack` was rejected because the current prerequisite and lab graph could not accept it. K2SO later rewrote that graph, but MIR had already reduced every generated stream to automation science.

The exact pre-fix load order was MIR, Krastorio 2, K2SO, then the local K2SO enhancement fork. The corrected order is Krastorio 2, K2SO, MIR, then the enhancement fork.

## Remediation

`info.json` now declares hidden optional dependencies on `Krastorio2` and `Krastorio2-spaced-out`. Hidden optional dependencies affect ordering only when those mods are present; they do not add a hard dependency or make K2 visible in the Factorio dependency UI.

Static validation requires both hidden dependencies and rejects visible or version-floored variants. The local-library scenario manifest now carries a positive `required_stream_science` contract. The compatibility audit reads the generated stream evidence and fails the scenario if any declared pack is absent.

For `research_advanced_circuit`, the exact K2SO scenario requires:

- `automation-science-pack`;
- `logistic-science-pack`;
- `chemical-science-pack`;
- `production-science-pack`;
- `electromagnetic-science-pack`.

The compatibility claim is now `partial-support`, backed by exact load and generated-science evidence rather than load success alone. Obsolete expected failures for invalid science and missing prerequisites were removed.

## Exact Runtime Result

The final exact-package scenario ran on Factorio 2.1.11 with eight resolved third-party mods and four official mods. It exited successfully, produced 5,557 audit rows, and reported zero dependency-resolution failures. The generated `research_advanced_circuit` row contained all five required packs with no missing science assertion. The grouped scenario audit classified 516 findings as expected and zero as unexpected.

The retained machine contract reports `science_contract_passed = true`. Large disposable runtime trees are intentionally deleted after their compact result is recorded so local qualification does not accumulate duplicate Factorio assets and mod archives.
