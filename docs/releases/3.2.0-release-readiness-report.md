# MIR 3.2.0 Release Readiness Report

## Verdict

MIR 3.2.0 completed a historical local automated qualification checkpoint. The development candidate recorded below passed all 125 tests then declared, including exact-package runtime lanes and the exact 3.1.9 upgrade proof. Release-boundary remediation after that checkpoint changes packaged source and the governed test set, so the recorded ZIP and 125-test result are no longer the final candidate or current full-profile result.

MIR 3.2.0 is **not release-qualified, sealed, frozen, tagged, or published**. Local evidence is intentionally ineligible for a release seal. A new deterministic candidate, exact approved delta, Factorio-version qualification, paired runtime performance comparison, package-focused manual attestation, and protected-release evidence remain mandatory.

## Release-Boundary Remediation

The governed implementation now closes the mechanically enforceable defects found after the checkpoint:

- the canonical full profile contains direct F4 tests for the exact approved delta, paired runtime performance evidence, and package-focused manual review;
- full and backport sealing requires passing performance and manual evidence and seal schema 4 binds their paths, hashes, and statuses;
- approved-delta validation accepts the active candidate and source authority and checks archive, content, and source identities;
- the shared effect contract covers quality and entity targets, quality-aware `give-item` identity, and all target-bearing modifiers in the governed Factorio 2.1.11 profile;
- `TechnologyDesign` consumes the shared effect-target projection instead of scanning a second target-field list;
- MIR no longer performs its own global `reset_technology_effects()` call; the headless configuration-change fixture proves unrelated force recipe state is preserved, while the required connected-player manual item proves the external `give-item` is not duplicated;
- the protected workflow transfers the exact candidate once and merges per-worker capsule deltas into a ledger restored only at the gate.

These implementation statements are not test results. The revised plan and runtime fixtures still need execution against the new candidate.

## Factorio Version Boundary

The revised Factorio 2.1 profile declares 2.1.11 as the primary qualification binary and 2.1.8 as the advertised minimum. No new 2.1.8 floor smoke or 2.1.11 protected qualification is recorded here. The dependency floor must not remain at 2.1.8 unless exact base and Space Age minimum-version smoke evidence is added; otherwise raise it to the oldest qualified patch before final packaging.

## Release And Backport Decision

Source freeze, tag, publication, and the MIR 2.5.0 backport remain blocked. The next authorized sequence is:

1. finish release text, build the deterministic replacement candidate, and record its descriptor;
2. regenerate the seven-scenario approved delta against exact 3.1.9 and replacement 3.2.0 bytes;
3. publish the commit-bound upgrade matrix and qualify Factorio 2.1.8 and 2.1.11 honestly;
4. run the paired 3.1.9-to-3.2.0 performance campaign and write the exact evidence;
5. complete and self-hash the package-focused manual attestation without changing package bytes;
6. run the final protected full profile and create and inspect the schema-4 seal;
7. freeze canonical 3.2 package source and release-evidence commits separately;
8. only then begin the independent 2.5.0 backport campaign.
