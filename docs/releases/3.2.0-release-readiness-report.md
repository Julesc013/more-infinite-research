MIR 3.2.0 has completed its local automated qualification checkpoint. The exact development candidate passed all 125 declared F0-F4 checks, including exact-package runtime lanes and the exact 3.1.9 upgrade proof.

## Performance Boundary

That comparison remains an explicit pre-seal automation gap under the 20 percent ceiling in `.mir/performance.yml`.

## Release And Backport Decision

Source freeze, tag, publication, and the MIR 2.5.0 backport remain blocked. The next authorized sequence is:

1. supply the qualified 3.1.9 runtime performance baseline and bind the comparison;
2. obtain the missing ecosystem archives if those claims are required for 3.2.0;
3. complete maintainer review;
4. run the final protected full profile and create the seal;
5. freeze canonical 3.2 source;
6. only then begin the independent 2.5.0 backport campaign.
