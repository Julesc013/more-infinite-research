## Release And Backport Decision

1. freeze and register the final package source and archive identities in `.mir/releases.json`;
2. obtain green hosted Validate and Branch Policy workflows, freeze the assurance contract, and materialize the definitive full plan;
3. complete every fresh exact-candidate automated lane, including exact-ZIP, approved-delta, upgrade, ecosystem, and performance work;
4. complete and self-hash the package-focused manual attestation without changing package bytes;
5. run the final protected full profile and create and independently inspect the schema-4 seal;
6. record canonical 3.2 package-source and release-evidence commits separately;
7. complete the separate public-presentation review and promote the sealed bytes without rebuilding;
8. only then begin the independent canonical 2.5.0 backport campaign.
