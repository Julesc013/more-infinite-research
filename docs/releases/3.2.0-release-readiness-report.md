# MIR 3.2.0 Release Readiness Report

## Verdict

MIR 3.2.0 completed a historical local automated qualification checkpoint. The development candidate recorded below passed all 125 tests then declared, including exact-package runtime lanes and the exact 3.1.9 upgrade proof. Release-boundary remediation after that checkpoint changes packaged source and the governed test set, so the recorded ZIP and 125-test result are no longer the final candidate or current full-profile result.

MIR 3.2.0 is **not release-qualified, sealed, frozen, tagged, or published**. Local evidence is intentionally ineligible for a release seal. Closing the 2.4.x line added its published changelog history to the packaged source and therefore produced a new development candidate. Exact-ZIP, approved-delta, upgrade, Factorio-version, paired runtime performance, package-focused manual, and protected-release evidence must bind these current bytes.

## Release And Backport Decision

Release seal, tag, publication, and the canonical MIR 2.5.0 backport remain blocked. The remaining authorized 3.2 sequence is:

1. qualify Factorio 2.1.8 and 2.1.11 honestly, or raise the declared floor before changing package bytes again;
2. run the paired 3.1.9-to-3.2.0 performance campaign and write the exact evidence;
3. complete and self-hash the package-focused manual attestation without changing package bytes;
4. run the final protected full profile and create and inspect the schema-4 seal;
5. record canonical 3.2 package-source and release-evidence commits separately;
6. complete the separate public-presentation review and promote the sealed bytes without rebuilding;
7. only then begin the independent canonical 2.5.0 backport campaign.
