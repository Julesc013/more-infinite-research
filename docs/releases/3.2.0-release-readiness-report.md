# MIR 3.2.0 Release Readiness Report

## Verdict

MIR 3.2.0 completed a historical local automated qualification checkpoint. The development candidate recorded below passed all 125 tests then declared, including exact-package runtime lanes and the exact 3.1.9 upgrade proof. Release-boundary remediation after that checkpoint changes packaged source and the governed test set, so the recorded ZIP and 125-test result are no longer the final candidate or current full-profile result.

MIR 3.2.0 is **not release-qualified, sealed, tagged, or published**. Local evidence is intentionally ineligible for a release seal. The current candidate has completed exact local Factorio 2.1.11 runtime, approved-delta, upgrade, ecosystem, and paired-performance qualification. Exact Factorio 2.1.8 floor evidence or a dependency-floor change, package-focused manual review, protected qualification, and sealing remain required.

## Release And Backport Decision

Release seal, tag, publication, and the canonical MIR 2.5.0 backport remain blocked. The remaining authorized 3.2 sequence is:

1. qualify Factorio 2.1.8 base and Space Age or raise the declared floor before changing package bytes again;
2. complete and self-hash the package-focused manual attestation without changing package bytes;
3. run the final protected full profile and create and inspect the schema-4 seal;
4. record canonical 3.2 package-source and release-evidence commits separately;
5. complete the separate public-presentation review and promote the sealed bytes without rebuilding;
6. only then begin the independent canonical 2.5.0 backport campaign.
