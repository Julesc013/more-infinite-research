# MIR 1.6.0 Lower-Wave Release Dossier

MIR 1.6.0 is a canonical MIR 3.1.9-derived target projection for Factorio 0.16, anchored at `7ca81bbc98f02cfaf2d40012096d94e261e74d98`. It is feature-complete for positively supported target capabilities, not identical to MIR 3.1.9 and not full modern parity.

## Player Scope And Limitations

The release provides the target-supported native direct-effect subset, old-science lowering, target-era settings and locale, compact formulas, conditional weapon ownership, iterative graph validation, and `global` runtime state. Recipe productivity, Space Age, quality, recycler, cargo-only effects, modern prototype limits, settings profiles, and unsupported scripted effects are omitted by capability. Manual visual, tree-layout, locale-fit, and human balance review remain `PENDING-MAINTAINER`.

## First-Release Retention

There is no legitimate prior same-line release, so no upgrade predecessor is invented. Qualification creates target state, reloads the exact candidate, mutates supported settings and research state, and proves their retention through the exact 0.16 binary.

## Feature, Settings, Locale, And Balance References

The no-blank-cell feature matrix is `.mir/evidence/lower-wave/1.6.0-feature-classification.json`. Only settings with a supported emitted effect are visible. The package carries English plus fallback-aligned locale catalogs and blocks malformed target parsing. `.mir/evidence/lower-wave/1.6.0-balance-summary.json` records deterministic costs, effects, bounds, science reachability, and ownership invariants; human balance acceptance remains pending.

## Source Lock, Seal, And Release Checklist

`.mir/backport-source-lock.json` binds the canonical anchor, completed predecessor ring, target profile, canonical feature model, adapted paths, evidence, and seal. `.mir/evidence/1.6.0-candidate-seal.json` binds the clean source commit, exact ZIP and content hashes, binary, harness, test catalog, fixtures, and qualification. Tagging is blocked until static, exact ZIP, reload, first-release retention, locale, balance, deterministic rebuild, source-lock, clean-state, and seal gates pass. Publication must upload the sealed bytes without replacing them, then publicly re-download and hash them. Portal status is `blocked-missing-MOD_UPLOAD_API_KEY` unless the credential exists.
