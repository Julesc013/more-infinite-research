# 3.2.0 Technology Design Hardening Report

## Outcome

This checkpoint hardens the first stream `TechnologyDesign` vertical slice without intentionally changing emitted technology behavior. It separates reusable design identity from context-specific qualification, makes the documented schema enforceable, requires the IR at the emitted GenerationPlan boundary, extends exact output parity to presentation, and bounds graph and sanitation evidence.

This remains a stream-only boundary. Native-owner adoption and base continuations do not yet use `TechnologyDesign`, and this report makes no promotion, broad automatic-generation, ecosystem, release, or backport claim.

## Enforced schema

`TechnologyDesign` is now schema 2. One Lua authority declares dimensions, typed subject categories, maturity enums, evidence classes, lock states, lock policies, required leaf paths, and cross-field invariant names. Runtime validation checks required structures, enum membership, identity authority, top-level and dimension consistency, leaf-provenance consistency, lock aggregation, members-to-subject projection, and every canonical fingerprint.

The validator now distinguishes `subject_fingerprint`, `design_fingerprint`, `prototype_fingerprint`, and `qualification_fingerprint`. The old `semantic_fingerprint` remains only as a compatibility alias for qualification. Compilation planning separately emits `compilation_fingerprint`, `qualification_fingerprint`, and `run_fingerprint`.

## Locks, identity, and subjects

Lock state is `none`, `partial`, or `all`; leaf records are authoritative. Adaptive policies that claim reviewed bounds carry machine-readable envelopes. `diff`, `assert_locks`, and `merge` reject unauthorized locked changes and changes outside an envelope.

Identity stability comes from explicit stream or candidate authority rather than a non-empty proposed name. Existing fixed streams retain released identity through the legacy stream manifest, and the two predeclared 3.1 automatic-family identities now explicitly declare `released`. A future automatic candidate defaults to provisional.

Typed subjects include recipes, products, items, fluids, entities, technologies, effect targets, science packs, and surfaces. Direct-effect capability preserves the actual modifier type or deterministic multi-type identity.

## Plan and parity boundary

Every `emit` row in GenerationPlan schema 3 must contain a validated TechnologyDesign schema-2 record. The legacy `fields` table is checked as a compatibility projection and cannot disagree. Compilation planning, stream emission, and GenerationPlan output validation consume the IR projection.

Canonical graph, prototype, presentation, and save-identity projections are separate. Output parity now checks localized name, localized description, icons, order, level, and declared enabled, hidden, and upgrade values in addition to effects, prerequisites, science, cost, time, and maximum level.

## Graph and sanitation evidence

An SCC now carries both a stable member-set identity and a topology fingerprint over sorted internal edges. Rejection propagation records causes once in a cause graph, keeps one deterministic primary reason, and emits only bounded contributing samples with counts and an origin fingerprint.

Sanitation evidence now fingerprints the full original effect list, each removed effect, retained semantic identities, the retained list, and the sanitized list. The input and output passes also bind every sanitation-covered target prototype family; postconditions fail if MIR creates or removes one of those targets after input sanitation. The input command is classified as `sanitation`, not emission.

## Compiler evidence transport

CompilerEvidence schema 2 is a target-neutral domain object. Factorio 2.1 publishes it with a `mod-data` adapter. Reduced targets have a validation-log adapter over the same compilation, qualification, run, sanitation, and evidence fingerprints.
