# MIR 2.4.0 Validation Summary

## Current Result

MIR 2.4.0 is a fresh Factorio 2.0 port of the accepted 3.1.0 source anchor. It retains the supported compiler contracts and removes the two Factorio 2.1-only cargo modifier streams. It is not a metadata-only downgrade and does not claim byte or feature parity with 3.1.0.

Evidence: `.mir/evidence/2.4.0-release-targeted-summary.json`.
- Exact MIR 2.3.5 upgraded to the 2.4.0 candidate with non-default scripted enable/effect settings, technology level, namespaced storage, and runtime multiplier retained. Evidence: `.mir/evidence/2.4.0-upgrade-proof.json`.

## Target-Specific Repairs

- Synthetic science-pack fixtures derive the active prototype type because Factorio 2.0 science packs are `tool` prototypes while Factorio 2.1 uses ordinary items.
- Recycler boundary recipes use Factorio 2.0's singular `category` field instead of the Factorio 2.1 `categories` field.
- Descriptor and settings assertions use the positive target profile so the two unsupported cargo streams are excluded without weakening the remaining contracts.
- Runtime validation dispatches Space Age and scripted gates from declared target capabilities instead of a hard-coded Factorio 2.1 branch check.

## Remaining Boundary

The exact final archive still requires a clean freshness record, interactive save/settings inspection, and publication promotion. External upload state is not inferred from a local archive or tag.
