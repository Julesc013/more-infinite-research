# MIR 3.1.9 Migration Guide

MIR 3.1.9 keeps the stable setting IDs, generated technology IDs, migrations, and runtime-state model from MIR 3.1.5. No manual save migration is required.

Existing non-default settings for processing units, plastic, low-density structures, and rocket fuel become effective when a recognized infinite owner exists. Review those six controls before loading an important save if they were previously changed while ineffective. Defaults preserve the existing owner exactly, and disabling the MIR stream performs no owner mutation.

Cost base and growth are one cost-model pair. Changing either applies both values shown in settings. During the upgrade or a later startup-setting change, MIR preserves the level, current selection, and fractional progress of a bound owner that is actively being researched.

For an owner supplied by another mod, default preservation remains safe even when MIR does not recognize the cost formula. An explicit base-cost or growth change requires a recognized formula and otherwise fails closed with a diagnostic. Restore the cost controls to their defaults or ask the owner mod to expose a supported formula.

The release gate covers exact upgrades from MIR 3.1.5, MIR 3.1.2, and MIR 3.0.5. Keep a normal save backup before changing any startup setting.
