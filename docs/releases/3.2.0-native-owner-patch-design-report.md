# 3.2.0 Native-Owner Patch Design Report

## Outcome

Native productivity-family adoption now uses schema-2 `TechnologyDesign` as its patch-existing authority. Every accepted adoption row carries the immutable pre-mutation owner fingerprint, the complete expected owner snapshot, the expected output fingerprint, the owner target, the adoption operation, and the explicit configured fields.

GenerationPlan rejects an adoption row without a valid design or when its projection, target, operation, configured fields, or output fingerprint differs from the transaction. CompilationPlan repeats the assertion when it materializes the operation. The emission-owned transaction checks the live input fingerprint and the design projection before applying the new effect. Output validation consumes the same design after mutation.

The patch design remains inside compiler planning artifacts. It is not added to the public generated-technology registry or the existing runtime adoption namespace, preserving the 3.1.9 transport and runtime contract.

## Validation

Architecture lint, schema-drift checks, and the compiler-contract mutation fixture passed. The fixture proves that target tampering cannot be made valid by merely recomputing the legacy row fingerprints.

Fresh exact-candidate Factorio 2.1.10 loads passed for `approved-delta-native-owner-adoption` and `space-age-vanilla-family-adoption`. The complete normalized seven-scenario comparison passed with 25 intentional differences, zero unapproved differences, and zero technology-field differences. The final deterministic Optimal archive SHA-256 is `A9AE21874C225B257D973C11D08CF811921B4D2F3E18496069BC7398FBCA2D89`.

## Boundary

This completes the common-IR migration for emitted streams, base continuations, and native-owner adoption. It does not itself create candidate, qualification, approval, promotion, or migration record types, and it does not broaden public compatibility claims.
