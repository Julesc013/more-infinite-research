# 3.1.0 Roadmap

Use 3.1.0 for behavior expansion after the 3.0 governance, manifest, docs, and
validation boundaries are stable.

## Candidate Work

Broaden report-first capabilities into propose-first or generated behavior only
after fixture evidence exists:

- loader manufacturing;
- mining-drill manufacturing;
- native modifier ownership;
- science and lab compatibility.

Continue staged compiler work without weakening the 3.0 package boundary:

- keep active implementation under `prototypes/mir/`;
- keep stream manifests stable;
- keep compatibility overlays declarative;
- add richer report diffing and claim export tooling.

## Later Waves

Use 3.2-style work for graph intelligence:

- recipe graph;
- resource-chain graph;
- technology graph;
- loop-risk strongly connected component detection;
- catalyst, container, filter, recycling, and voiding detection.

Use 3.3+ work for fixture-backed compatibility campaigns:

- AAI Loaders;
- Big Mining Drill;
- ATAN Nuclear Science;
- Krastorio 2;
- Bob's material families;
- Angel/Bob;
- Space Exploration 2.0 lane;
- Pyanodon smoke fixtures.
