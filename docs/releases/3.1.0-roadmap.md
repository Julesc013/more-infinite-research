# 3.1.0 Roadmap

Use 3.1.0 for behavior expansion after the 3.0 governance, manifest, docs, and
validation boundaries are stable.

## Candidate Work

### Compiler Contracts

- Replace target denylists with declaration-owned positive feature
  requirements and validate every stream, setting, effect, pass, and runtime
  handler against every maintained target.
- Load immutable canonical stream descriptors, apply compatibility patches to a
  clone, run final target filtering, reject duplicate IDs, then validate.
- Move toward one declarative stream descriptor that generates setting IDs,
  defaults, ordering, effect contracts, target requirements, science metadata,
  documentation rows, and expected validation cases.
- Split science facts from selection policy around one immutable fact graph
  shared by prerequisite, researchability, lab, owner, and graph-safety checks.
- Either complete data-driven capability resolver dispatch or reduce the
  framework to focused provider interfaces before exposing any public API.

### Performance And Reproducibility

- Build immutable recipe-productivity, native-effect, and researchability owner
  indexes and reuse them across policy, adoption, competition, diagnostics, and
  cleanup.
- Split the validation runner into scenario setup, settings override, Factorio
  process, assertion, and evidence modules without changing its CLI.
- Replace source-text fixture overrides with declarative test settings.
- Introduce deterministic ZIP construction with sorted entries, fixed
  timestamps, normalized paths/permissions, fixed compression, and a two-clean-
  worktree identical-SHA proof.

Broaden report-first capabilities into propose-first or generated behavior only
after fixture evidence exists:

- loader manufacturing;
- mining-drill manufacturing;
- native modifier ownership;
- science and lab compatibility.

Continue staged compiler work without weakening the 3.0 package boundary:

- keep active implementation under `prototypes/mir/`;
- keep stream manifests stable;
- keep compatibility overlays declarative;
- add richer report diffing and claim export tooling.

## Later Waves

Use 3.2-style work for graph intelligence:

- recipe graph;
- resource-chain graph;
- technology graph;
- loop-risk strongly connected component detection;
- catalyst, container, filter, recycling, and voiding detection.

Use 3.3+ work for fixture-backed compatibility campaigns:

- AAI Loaders;
- Big Mining Drill;
- ATAN Nuclear Science;
- Krastorio 2;
- Bob's material families;
- Angel/Bob;
- Space Exploration 2.0 lane;
- Pyanodon smoke fixtures.
