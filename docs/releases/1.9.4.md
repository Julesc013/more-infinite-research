# MIR 1.9.4 Factorio 1.1 Release Record

MIR 1.9.4 is a canonical MIR 3.1.9-derived target projection, feature-complete for the capabilities supported by the target Factorio engine. It is projected from qualified development anchor `93972f4ea980d69652df89ebafea94828dcf0113`; historical 1.9.x commits are behavioral evidence, not source authority.

## Scope

- Target: Factorio 1.1, with `base >= 1.1` as the only required dependency.
- Stable surface: eleven generated research streams, legacy `tool` science-pack discovery, supported direct effects, conditional weapon-overlap policy, startup settings, and `global` runtime state.
- Omitted surface: modern recipe-productivity compilation, Space Age and 2.x-only prototype controls, settings profiles, scripted Space Age effects, modern compatibility repairs, and native-owner adoption.
- Claims: only the named exact-binary scenarios in the qualification summary support this release. No broad third-party compatibility claim is implied.

## Upgrade And Settings

The qualification campaign must prove a clean load, a reload, and an upgrade from the released 1.9.3 archive. Existing generated technology IDs remain manifest-backed. The supported settings surface is the reduced Factorio 1.1 startup set; settings-profile import/export and modern pipeline/prototype-limit controls are not part of this release.

## Release Gate

Publication requires a clean source-lock check, static validation, two byte-identical package builds, the exact Factorio 1.1.110 binary, a fresh candidate seal, and successful post-upload verification. The final archive hash and publication URLs are recorded only after those gates pass.
