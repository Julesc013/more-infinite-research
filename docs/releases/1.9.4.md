# MIR 1.9.4 Factorio 1.1 Release Record

MIR 1.9.4 is a canonical MIR 3.1.9-derived target projection, feature-complete for the capabilities supported by the target Factorio engine. ## Source And Scope

- Target: Factorio 1.1, with `base >= 1.1` as the only required dependency.
- Canonical released source: MIR 3.1.9 tag commit `1b9c6f32fc2bb53c413a593534e103a6043b4be3`, package source `79df29b50ea9b855b8665d6c9a3d8295806acde0`.
- Target qualification source: `fc1a5b0bb56b11e877eb69272a7904d991cf4cea`.
- Stable surface: eleven generated research streams, legacy `tool` science-pack discovery, supported direct effects, conditional weapon-overlap policy, startup settings, and `global` runtime state.
- Omitted surface: modern recipe-productivity compilation, Space Age and 2.x-only prototype controls, settings profiles, scripted Space Age effects, modern compatibility repairs, and native-owner adoption.
- Claims: only the named exact-binary scenarios in the qualification summary support this release. No broad third-party compatibility claim is implied.

The complete feature and omission matrix is `.mir/evidence/1.9.4-feature-classification.json`; `.mir/backport-source-lock.json` records every portable, adapted, target-specific, and capability-omitted surface.

## Settings And Upgrade Guide

The supported settings surface is the reduced Factorio 1.1 startup set. Settings-profile import/export and modern pipeline/prototype-limit controls are absent rather than inert. Fresh installs use conditional rocket/cannon overlap cleanup: vanilla categories are removed only after a valid infinite replacement owner is accepted. Explicit `off` and `always` values are retained during upgrade.

Upgrade qualification used the exact released `more-infinite-research_1.9.3.zip` (`1723C10FEDD9A12003052991CC7574F1F6BF4E4ABC506F0323571DF680C0444B`). Both retained-setting paths passed, including generated technology level, current research, fractional progress `0.42`, and `global.mir` runtime state. Evidence is under `.mir/evidence/1.9.4/upgrade-off` and `.mir/evidence/1.9.4/upgrade-always`.
