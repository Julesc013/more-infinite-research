# 3.2.0 Compiler Context Ownership Report

## Outcome

Mutable compiler state derived from prototypes, settings, active mods, or the current data-final-fixes pass is now owned by CompilerContext schema 2. The refactor removes process-wide `canonical`, `latest`, and ledger tables from the production compiler path without intentionally changing generated technologies or public runtime contracts.

The context owns recipe and relationship indexes, active compatibility packs, family resolution, GenerationPlan and CompilationPlan artifacts, generated-technology registration, native-owner adoption evidence, coverage, telemetry, and diagnostics. Input and output sanitation ledgers remain immutable context artifacts. Built-in provider and FamilyRule catalogs remain module-scoped because they are immutable policy sources.

## Isolation contract

Creating a context activates it for compiler modules that consume the current Factorio data-stage run. Explicit command execution reactivates its supplied context. Named state has an internal read-only-by-contract view for one run and a deep-copy snapshot for publication or tests. The compiler-contract fixture proves that two contexts do not share derived state or immutable artifacts and that mutating a context snapshot cannot mutate its owner.

Architecture lint rejects the former module-level caches in the affected index, resolver, compatibility, planner, and report modules. It also requires every context-owned module to declare its CompilerContext dependency.

## Validation

The context-owned compiler passed architecture lint and the exact-package `base-generation-integrity` Factorio 2.1.10 load check. The refactored archive was byte-deterministic with SHA-256 `313F712B659C3F49C0A9D1C4E8D398D5854AD90262AA717BFD2E3F541E09773A`.

The normalized approved-delta artifact was regenerated through fourteen fresh Factorio 2.1.10 data stages: seven scenarios for the sealed 3.1.9 archive and the same seven for the context-owned compiler. It retained 24 intentional contract/package differences, zero unapproved differences, and zero normalized technology-field differences. Base-derived scenarios retained 52 technology identities and Space Age-derived scenarios retained 70.

This checkpoint does not promote 3.2.0, broaden automatic generation, claim ecosystem compatibility, or authorize backports.
