# MIR 3.1.0 Release Checklist

## Entry And Characterization

- [x] Qualify, freeze, integrate, and tag exact 2.3.5; keep external upload status explicit.
- [x] Create `pre-3.1-modernization` from the accepted source anchor.
- [x] Record 3.0.5 generated IDs, settings defaults, profiles, runtime namespaces, planner rows, and package identity.
- [x] Confirm no target-local metadata or feature cuts are present on `dev`.

## Architecture

- [x] Explicit typed primary effect metadata is complete and order-invariant.
- [x] One immutable normalized recipe fact authority serves all supported fact consumers; mutators and recycler preflight remain explicit phase-local operations.
- [x] Science integration is split around shared facts and reason codes.
- [x] Every stream, setting, effect, pass, handler, and governed fixture declares positive target requirements.
- [x] Canonical descriptors are cloned after raw overlays and cannot be mutated through `require()` caches.
- [x] Duplicate IDs and overlay attempts to recreate or inject filtered declarations fail.
- [x] Prototype mutation is limited to declared pipeline commands, emitters, and exact-version repair exceptions.
- [x] `.mir/modules.yml`, ADRs, architecture docs, and lints match the implementation completed so far.
- [x] All fixed streams compile through GenerationPlan schema 2 before emission.
- [x] RecipeFactV2 and shared relationship indexes preserve variant and ownership evidence.
- [x] FamilyRule and CompatibilityPack schema 2 are data-only and fail closed.
- [x] Complete recipe accounting and auditable graph/scan counts are emitted.

## Correctness And Tests

- [x] All 3.0.5 scenarios remain green with no unexplained generated-ID change.
- [x] Recycler boundary corpus covers multiple paths, byproducts, probability, ignored productivity, catalysts, fluids, variants, conversions, and nonstandard returns.
- [x] Unrestricted module cross-policy diagnostics have explicit outcomes.
- [x] Scripted effects cover multiple levels, reversal, reset, configuration changes, force lifecycle, save/load, disable/enable, rebasing, and restoration.
- [x] Profile tests cover every effect setting, compact omission, numeric bounds, and mixed target visibility.
- [x] Dependency-cycle reachability, catalog immutability, mutation-boundary, and large synthetic graph tests pass.

## Tooling, Packaging, And Performance

- [x] Validation is decomposed while preserving its CLI and result schema.
- [x] Scenario declarations and settings overrides are declarative and target-explicit.
- [x] Two clean worktrees build byte-identical archives.
- [x] Base, Space Age, scoped caps, diagnostics, medium pack, and large pack timings are recorded against declared budgets.
- [x] Repeated full recipe scans inside per-item loops are rejected; technology scan budgets remain part of the performance gate.

## Capability Slice

- [x] Loader manufacturing attaches structurally proven recipes to the existing belt stream and records diagnostics.
- [x] Mining-drill manufacturing attaches structurally proven recipes to the existing mining-drill stream.
- [x] Reviewed assembler and lab family identities emit only under `safe-generate`; default `safe-attach` preserves the conservative boundary.
- [x] Native modifier ownership and science/lab hard gates use the shared facts and positive requirements.
- [x] Named mods remain fixtures, not architecture branches.

## Release Gates

- [x] Rerun the complete static and Factorio 2.1 matrix after the planet-cluster prerequisite correction (`86/86` at `3699fec`).
- [x] Run the exact 3.0.5-to-3.1.0 upgrade with settings, research levels, persisted state, and scripted effects retained.
- [x] Load the exact tested 3.1.0 archive from isolated normal mods directories in base and Space Age configurations.
- [x] Bind independent available-ecosystem campaigns to the exact archive and source commit at honest `loads` claim level (`9/9`, including a 46-mod planet cluster).
- [ ] Review balance, technology visibility, settings, locale, and icons interactively on the final candidate.
- [x] Refresh release notes, migration guide, validation summary, hashes, and candidate freshness from the clean RC3 tree.
- [ ] Publish and freeze the exact validated bytes only after every gate passes.
