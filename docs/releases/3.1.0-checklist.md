# MIR 3.1.0 Release Checklist

## Entry And Characterization

- [x] Qualify, freeze, integrate, and tag exact 2.3.5; keep external upload status explicit.
- [x] Create `pre-3.1-modernization` from the accepted source anchor.
- [x] Record 3.0.5 generated IDs, settings defaults, profiles, runtime namespaces, planner rows, and package identity.
- [x] Confirm no target-local metadata or feature cuts are present on `dev`.

## Architecture

- [ ] Explicit typed primary effect metadata is complete and order-invariant.
- [ ] One immutable normalized recipe fact authority serves all supported consumers.
- [ ] Science integration is split around shared facts and reason codes.
- [ ] Every stream, setting, effect, pass, handler, and fixture declares positive target requirements.
- [ ] Canonical descriptors are cloned before overlays and cannot be mutated through `require()` caches.
- [ ] Duplicate IDs and overlay attempts to recreate filtered declarations fail.
- [ ] Prototype mutation is limited to declared pipeline commands, emitters, and exact-version repair exceptions.
- [ ] `.mir/modules.yml`, ADRs, architecture docs, and lints match the implementation.

## Correctness And Tests

- [ ] All 3.0.5 scenarios remain green with no unexplained generated-ID change.
- [ ] Recycler boundary corpus covers multiple paths, byproducts, probability, ignored productivity, catalysts, fluids, variants, conversions, and nonstandard returns.
- [ ] Unrestricted module cross-policy diagnostics have explicit outcomes.
- [ ] Scripted effects cover multiple levels, reversal, reset, configuration changes, force lifecycle, save/load, disable/enable, rebasing, and restoration.
- [ ] Profile tests cover every effect setting, compact omission, numeric bounds, and mixed target visibility.
- [ ] Dependency-cycle reachability, catalog immutability, mutation-boundary, and large synthetic graph tests pass.

## Tooling, Packaging, And Performance

- [ ] Validation is decomposed while preserving its CLI and result schema.
- [ ] Scenario declarations and settings overrides are declarative and target-explicit.
- [ ] Two clean worktrees build byte-identical archives.
- [ ] Base, Space Age, scoped caps, diagnostics, medium pack, and large pack timings are recorded against declared budgets.
- [ ] Repeated full recipe or technology scans inside per-item loops are rejected.

## Capability Slice

- [ ] Loader manufacturing completes the full lifecycle from structural discovery through emission/adoption and diagnostics.
- [ ] Mining-drill manufacturing completes the same lifecycle.
- [ ] Native modifier ownership and science/lab hard gates use the shared facts and positive requirements.
- [ ] Named mods remain fixtures, not architecture branches.

## Release Gates

- [ ] 3.0.5-to-3.1.0 upgrade passes with settings and runtime state retained.
- [ ] Full Factorio 2.1 base, Space Age, and local-library suites pass.
- [ ] Exact candidate archive loads from a normal mods directory.
- [ ] Release notes, migration guide, architecture docs, changelog, and validation summary are current.
- [ ] Candidate freshness passes from a clean tree.
- [ ] Interactive save and settings review passes.
- [ ] Publish and freeze exact bytes only after every gate passes.
