# MIR 2.4.9 Stability Backport

MIR 2.4.9 is a bounded emergency stability patch for the Factorio 2.0 `legacy` line. Its package source starts from published MIR 2.4.5 and carries the Factorio 2.0 `mod-data` capability guard, exhaustive technology-effect target sanitation, quality-aware `give-item` identity, entity lookup for `turret-attack`, pre-index and final sanitation phases, removal of the unproven global technology-effect reset, complete localization for all 50 Factorio languages, a bounded Space Exploration recipe-removal repair, and an explicit steel productivity stream.

## Publication Outcome

MIR 2.4.9 was published on GitHub on 2026-07-20 from tag commit `7ebe93029695bbf809a15a14c6540530738a9e62`. The immutable release archive is `dist/more-infinite-research_2.4.9.zip`, SHA-256 `B5503F94D04624F65462CC275FB6AA71A8CE93075F732DF498F6D73AD255F978`, with package-content SHA-256 `23D992943090BFF487675E9DF8C5C12BFDB1F3018B0BF04C9928265E5DC95255`.

The maintainer explicitly directed promotion and publication after the local automated campaign. Package-focused manual attestation and protected qualification were not recorded before publication and must not be described retroactively as passed. The release record preserves that assurance exception while freezing the published bytes.

## Compatibility Contract

The patch preserves every MIR 2.4.5 public setting ID, generated technology ID, migration, runtime namespace, and default generation choice. Its only new generated family is the declared stable steel productivity stream. It does not import the TechnologyDesign lifecycle, broaden automatic family discovery, or claim broad new ecosystem support. Those changes remain in the 2.5/3.2 roadmap. Copper, iron, and steel matching excludes scrap-input recovery recipes in both generated and native-owner paths.

The sanitation authority covers `change-recipe-productivity`, `unlock-recipe`, `unlock-space-location`, `give-item` item and quality targets, `ammo-damage`, `gun-speed`, `unlock-quality`, and `turret-attack`. Missing target effects are removed without reordering retained effects. Exact ecosystem campaigns must still treat an unexpected external prune as review-required evidence rather than silently upgrading a support claim.

Every shipped locale has exact key parity with English. Source-hash-bound translation memory and the locale gate reject stale source text, empty values, placeholder or rich-text drift, unsupported locale directories, excessive UI text, internal translation markers, and untranslated English prose before packaging.

## Release Gates

The exact MIR 2.4.5 archive is the behavioral and upgrade baseline. The exact MIR 2.4.9 candidate must pass the governed Factorio 2.0 backport plan, deterministic packaging, exact-ZIP base and Space Age lanes, the complete runtime matrix, the 2.4.5-to-2.4.9 upgrade regression, ecosystem qualification, exact candidate delta validation, paired runtime performance comparison, and package-focused manual review. The seal must bind the exact archive/content hashes and the performance and manual attestation hashes and passing statuses.

The paired comparison uses six fixed lanes: base, Space Age, the BZ suite as the medium ecosystem, a locally complete 19-root BZ/5Dim/AAI/ATAN/Dectorio/Flow Control stress closure as the large ecosystem, diagnostics off, and diagnostics on. The large closure is performance-only evidence, not a compatibility support claim. Each package receives one warm-up and at least five measured runs per lane in alternating within-pair order. Ecosystem results bind the resolved dependency names, versions, and archive hashes. Diagnostic settings are supplied by an external settings-stage fixture, leaving both published package archives byte-exact.

Local automated evidence proves implementation readiness but does not substitute for protected producer identity or maintainer visual review. The governed process required those F4 inputs before promotion; the recorded publication outcome above documents the maintainer-directed exception instead of converting missing attestations into passing evidence. The published archive must not be rebuilt.

The final local backport plan contains 107 tests. All 106 machine-verifiable tests pass, including 92 Factorio scenarios, exact upgrade, representative ecosystem, approved-delta, and runtime-performance gates. The only local failure is `manual.release-review`, because the maintainer-authored attestation for the exact candidate is intentionally absent. Protected qualification and sealing have not run.

The earlier localization-complete candidate with SHA-256 `52B93336AB24FAF2E626835EAF971310E6D919A99083994B6F777F45CEBCB162` is superseded because steel productivity and the Space Exploration repair change packaged behavior. The replacement archive became the immutable published package after completing the automated runtime, upgrade, approved-delta, and performance campaign; manual and protected gates remain recorded as not performed.

## Paired Performance Result

The final campaign used one warm-up and five measured runs for each package in every lane. Positive percentages indicate a slower 2.4.9 median; negative percentages indicate a faster median.

| Lane | MIR 2.4.5 median | MIR 2.4.9 median | Delta | Result |
| --- | ---: | ---: | ---: | --- |
| Base | 3.934327 s | 3.831128 s | -2.623041% | Passed |
| Space Age | 9.195780 s | 9.221930 s | +0.284370% | Passed |
| BZ medium ecosystem | 4.808338 s | 4.847707 s | +0.818765% | Passed |
| 19-root large ecosystem | 5.656977 s | 5.613328 s | -0.771596% | Passed |
| Diagnostics off | 4.134561 s | 4.185263 s | +1.226297% | Passed |
| Diagnostics on | 4.970161 s | 5.289437 s | +6.423856% | Passed |

The ecosystem lanes use the isolated Factorio process duration rather than compatibility-audit setup time. The largest positive median change is 0.319276 seconds in the diagnostics-on lane, which is a 6.423856% increase. It remains below the 20% regression ceiling and the declared absolute budget. Base and the 19-root large ecosystem were faster at their medians. The accepted evidence is `.mir/evidence/2.4.9-performance-regression.json`, SHA-256 `9A2F39C59E6F37BA416920F8D550216F65206E81D2D5520FD3ED95A4781ABC37`.

## Upgrade Regression

The upgrade fixture records native-owner settings, level, selected research, fractional progress, and fixture storage under MIR 2.4.5, then loads the exact MIR 2.4.9 archive. It also researches an external `give-item` technology and disables a fixture recipe after its unlock. The candidate must preserve that custom recipe state, proving that MIR did not perform a global technology-effect reset during configuration change.

## Deferred Work

MIR 2.5 will receive an independent source lock, 2.4.9-to-2.5.0 approved delta, performance campaign, upgrade matrix, manual attestation, and protected seal. MIR 3.2 remains the canonical Factorio 2.1 compiler-foundation line. Neither future scope is authorized to enter this emergency patch implicitly.
