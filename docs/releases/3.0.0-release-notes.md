# More Infinite Research 3.0.0 Release Notes

MIR 3.0.0 is the compatibility compiler architecture release for Factorio
`2.1`.

This release preserves generated technology IDs while moving active
implementation under the MIR compiler namespace. It removes the old shipped
compatibility/library/legacy helper paths, keeps compatibility overlays
declarative, and keeps generated-output behavior stable against the recorded
2.2 comparison lanes.

## Player-Facing Summary

- Existing generated technology IDs are preserved.
- No new migration is required solely because implementation moved into MIR
  namespaces.
- Existing `2.0.5` and `2.1.0` migrations remain shipped.
- Space Age remains optional.
- Quality, Recycler, and Elevated Rails remain optional or hidden optional
  dependencies as recorded in `info.json`.
- Scripted spoilage preservation and agricultural growth speed remain disabled
  by default.
- Compatibility claims remain narrow and evidence-backed; this is not a broad
  "full support for every overhaul" release.

## Architecture Summary

- Factorio root lifecycle files stay as thin wrappers.
- Active shipped implementation lives under `prototypes/mir/`.
- `prototypes/streams/` remains declarative stream data for `3.0.0`.
- Emission and platform adapters own generated technology mutation.
- Generated technology IDs are backed by the stream manifest.
- Compatibility policy overlays describe behavior instead of directly mutating
  prototypes.
- Diagnostics describe generated, skipped, rejected, and observed behavior.

## Known Non-Blockers

- Portal-backed full-catalog checks were not run in the release environment
  because `FACTORIO_TOKEN` was not set.
- Local supported-zip isolation found `atan-ash_2.2.1` and
  `atan-nuclear-science_0.3.3` failing without MIR on the tested Factorio
  `2.1` setup. Those are tracked as upstream package/schema issues, not MIR
  regressions.
