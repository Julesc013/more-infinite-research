# MIR 2.3.5 Release Notes

MIR 2.3.5 is the maintained Factorio 2.0 port of the MIR 3.0.5
compatibility-hardening source. It keeps the modern compiler architecture and
Factorio 2.0-supported recipe productivity, settings profiles, prototype
limits, runtime storage, and optional Space Age behavior.

## Fixes

- Science recipe unlocks now reject disabled, unreachable, self-locked, cyclic,
  or non-researchable technologies before generated prerequisites are emitted.
- Required technologies use the same researchability authority and unsafe
  streams skip with a diagnostic instead of creating a fatal graph edge.
- Breeding productivity excludes incineration sinks, including the reported
  Planet Crucible and Rigor Module late-removal lifecycle.
- Weapon overlap cleanup changes only MIR-generated continuations and requires
  exact, researchable replacement coverage.
- Finite vanilla weapon speed bonuses and explicit stored overlap choices are
  preserved.

## Target Boundaries

The package targets Factorio 2.0 with `base >= 2.0`, optional Quality, and
optional Space Age. Factorio 2.1-only cargo bay unloading distance and cargo
landing pad count modifiers remain excluded. Generated IDs for retained
streams and persisted-state schemas are unchanged from 2.3.0.
