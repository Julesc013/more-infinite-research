# 3.2.0 Scale And Order Invariance Report

MIR now has source-bound 100,000-scale compiler evidence for recipe facts, technologies, effects, and graph edges. Ordered and deterministically permuted inputs produce identical coverage, generation, compilation, and explicit-input fingerprints. The four exact-package Factorio 2.1.10 scenarios passed from clean source and validation-harness state at commit `3d4ee88f580fecf5bca9b962a22e42dc06ba0e63`.

This closes the development roadmap's missing scale and randomized-order checkpoint. It is not a release seal, an ecosystem claim, or a replacement for final 3.1.9 performance-regression qualification.

## Proof Surfaces

The graph campaign materializes 60,000 technologies, effects, and shallow edges in Factorio, below the engine's 65,535 technology prototype limit. Its assertion then sends 100,000 planned technologies with 100,000 effects and 100,000 edges through MIR's production iterative graph validator. The in-memory topology contains one 25,000-node SCC, 1,000 five-node SCCs, bounded 64-node samples and witnesses, and rejection propagation across the complete planned graph.

The recipe campaign materializes 1,000 structural family candidates, proving high-fanout discovery and one canonical production scan. Its assertion sends 100,000 explicit recipe prototypes through the same canonical index builder used by production compilation. Chunked fingerprints cover every normalized fact and all output, productive-output, ingredient, and category indexes without constructing one unbounded canonical string.

The deterministic permutation uses a bijection over each input domain. The validator runs ordered then permuted scenarios in one process and rejects any mismatch across the four fingerprint classes.

## Engine Boundaries Found

The initial combined fixture was not a valid acceptance shape. Combining 100,000 recipes with 100,000 technologies exceeded the 600-second lane before MIR completed. Splitting independent stress domains made the result attributable and bounded.

Factorio rejects more than 65,535 technology prototypes after the data stage. A later materialized 25,000-node technology SCC completed MIR's iterative validation and emitted the expected fingerprint, but Factorio's own recursive post-load cycle check stack-overflowed. The final governed fixture therefore keeps engine-owned prototypes below the hard ID limit and engine-owned topology shallow, while exercising the larger and adversarial topology through MIR's iterative validator. The report does not claim that Factorio itself supports a 100,000-prototype or 25,000-node cyclic technology graph.

This evidence proves deterministic MIR compiler behavior at the requested logical scale and a large legal Factorio materialization.
