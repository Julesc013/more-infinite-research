# MIR 1.5.0 Factorio 0.15 Plan

## Scope

MIR 1.5.0 is the first current Factorio 0.15 plan. It follows the accepted 1.6.0 contracts only after independent target proof. Archived 0.15 planning records with 1.8.x version slots remain historical and do not authorize implementation.

## Non-goals

- Do not assume Factorio 0.16 or modern recipe, science, effect, icon, or runtime schemas.
- Do not simulate unsupported recipe productivity.
- Do not claim parity with any newer MIR target.

## Required Changes

- Build an exact Factorio 0.15 target profile from the matching binary and base files.
- Declare the positive native-effect allowlist and semantic old-science role map.
- Adapt target-era technology, recipe, asset, locale, and runtime-state shapes.
- Reuse only portable schemas, deterministic IDs, plan records, owner checks, and graph safety that pass target validation.
- Fail closed for unsupported emitters and prototype fields.

## Validation

- Run static package and architecture checks on `tmp/0.15`.
- Load the exact candidate in the matching Factorio 0.15 binary.
- Prove every emitted effect, infinite formula or finite fallback, prerequisite, science ingredient, icon, and locale key.
- Run fresh-create and reload proof plus upgrade proof when a prior target save is available.
- Inspect the release ZIP for modern metadata or excluded development files.

## Compatibility Impact

This is a new reduced target line. It must not change modern generated IDs, settings, behavior, or frozen archives.

## Release Notes

Describe 1.5.0 as a reduced Factorio 0.15 edition and enumerate only binary-proven features.

## Rollback

Discard the unpublished candidate and return to the branch source anchor. Never replace a published archive.
