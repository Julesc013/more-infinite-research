## Implemented fixed-point contracts

### Technology decision chain

`TechnologyDesign` now enforces closed enums and cross-field invariants. Hard gates use explicit `not-applicable`, `pending`, `passed`, `failed`, and `superseded` records. Pending evidence produces a proposal; authoritative pass or failure binds evaluator identity and a canonical evidence fingerprint. Provisional evidence is superseded rather than overwritten invisibly.

`SafetyQualification`, `DesignAssessment`, and `PromotionAuthorization` are independent records. A safety pass cannot claim design quality, reviewed trust, identity stability, applicability, runtime action, or public compatibility. Reviewed generation requires an exact MIR-owned authorization reference with matching pack, family, provider, provider version, trust class, and evidence. External self-asserted trust remains non-promoting.

The schema-2 `TechnologyCatalog` is always built as the canonical alternative inventory. Its deterministic policy selects only qualified or proposal alternatives, excludes rejected alternatives, and is invariant under discovery order. The schema-3 GenerationPlan is checked as an exact projection of catalog selection.

### Graph semantics

The new shared `prototypes/mir/graph/` kernel owns graph snapshots, iterative strongly connected components, condensation, reachability, researchability, graph-wide researchability indexing, qualification, diff, and proof evidence. Planner and emitter modules are adapters over the same semantics. Virtual and realized validation bind exact graph, component-assignment, condensation-topology, and proof fingerprints before parity succeeds.

The synthetic scale fixture exercises 100,000 in-memory technologies, effects, and graph edges; a 25,000-node SCC; 1,000 smaller SCCs; bounded 64-node diagnostics; and 60,000 materialized Factorio technologies, effects, and graph edges below the engine limit.

### Compiler lifetime

CompilerContext schema 3 owns per-run services, state, artifacts, state epochs, and freeze status. Services are registered per context and may be frozen. Science modules no longer configure module-global callbacks. Derived state is set once unless an explicit replacement supplies the current epoch; frozen state rejects replacement. Snapshots are observationally pure deep copies and cannot mutate live telemetry.

A graph-wide researchability index replaces per-root prerequisite-closure caches. Structural SCC and prerequisite status is computed once per context; individual queries return transient reachable-name projections.

### Providers and budgets

Provider resolution is decomposed into explicit discovery, normalization, classification, pack-policy, hazard-policy, owner-arbitration, decision, and budget stages. Each stage returns copied data rather than mutating upstream inputs.

Budgets are scoped by provider, family, partition, and profile. They account for absolute candidates, attachments, review rows, new and unreviewed candidates, retained-baseline growth, progression span, and semantic clusters. Only `new-unreviewed` proposals may be suppressed. Exact-reviewed attachments remain eligible and hard-safety blockers remain visible diagnostics.

### Dependency ownership

Presentation construction moved to `prototypes/mir/presentation/icon_builder.lua`; the historical emit path is a compatibility facade. Pure blocked-effect policy moved to the domain layer. `.mir/module-dependencies.json` and `scripts/Test-MIRModuleDependencies.ps1` enforce forbidden domain/graph/planner-to-emitter imports, overlay mutation rules, command authority, required targets, and an exact bounded legacy-cycle baseline.

## Remaining release gates

Before C7 can be released, release engineering must freeze the qualification source and definitive plan, execute every required automated lane fresh, complete exact package and public-presentation human review, run protected qualification on the authorized self-hosted runner, create and independently check the schema-4 seal, tag the release-evidence commit, and publish the already-sealed ZIP without rebuilding.

Any later package-visible change creates C8. Non-package assurance or evidence changes remain C7 qualification revisions with a new qualification source and, when applicable, a new test-set or verifier fingerprint.
