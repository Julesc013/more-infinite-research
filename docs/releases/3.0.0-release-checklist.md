# 3.0.0 Release Checklist

- [ ] `.mir/` manifests are current.
- [ ] Docs lint passes.
- [ ] Static validation passes.
- [ ] Runtime fixture validation passes on the target Factorio line.
- [ ] Package hygiene validation proves repo-only folders are excluded.
- [ ] Package build targets `3.0.0`, not a refreshed published `2.2.0` archive.
- [ ] Compatibility claims match fixture or named load-check evidence.
- [ ] Generated stream changes have migration review.
- [ ] Changelog and README are updated for players.

## Version Bump Gate

Do not bump `info.json` to `3.0.0` until all of these are true:

- [ ] The active source package target is `3.0.0`; published
  `dist/more-infinite-research_2.2.0.zip` and legacy `1.9.2` archives remain
  unchanged.
- [ ] Root stage files enter MIR stage modules, and data-final-fixes does not
  primarily depend on old root-level orchestration.
- [ ] Generated technology prototypes emit through validated
  `StreamSpec -> technology_builder` paths.
- [ ] Emission uses the Factorio platform adapter rather than direct
  `data:extend` calls outside the emit boundary.
- [ ] Compatibility overlays for Air Scrubbing and ATAN Ash are declarative,
  loaded through `prototypes/mir/compatibility/`, and consumed by stream policy.
- [ ] DecisionRecord rows cover current generated, diagnostic, rejected, and
  observed compiler report surfaces.
- [ ] `.mir/streams.yml`, `prototypes/mir/streams/generated_stream_manifest.json`,
  and compatibility claim registries agree for every stable generated stream.
- [ ] Architecture lint fails on new direct prototype mutation outside allowed
  emission or legacy-transition paths.
- [ ] Static validation, runtime fixture validation, package hygiene, and
  regression row/hash comparison pass or have documented intentional diffs.
- [ ] Backport notes identify the later `2.3.0` and `1.9.3` target-line work
  without changing already-published `2.2.0` or `1.9.2` artifacts.
