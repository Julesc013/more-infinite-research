# MIR 2.4.1 Validation Summary

## Current Result

MIR 2.4.1 is being qualified from the published 2.4.0 Factorio 2.0 source baseline. The package-visible changes are the 2.4.1 identity and changelog; the shared prerequisite-reachability implementation is unchanged because 2.4.0 already contains the portable transitive reachability fix.

The new regression disables the vanilla `automation-science-pack` technology, explicitly leaves the corresponding recipe initially available, and then requires every MIR-generated technology to retain an enabled, acyclic, reachable prerequisite graph. This reproduces the reported 2.3.5 configuration without weakening MIR's strict terminal graph assertion.

Exact candidate identity and final automated results will be recorded here after the focused Factorio 2.0 validation and deterministic package build complete.

## Remaining Boundary

Interactive review, tagging, GitHub release creation, and Mod Portal upload remain maintainer-only gates and are not performed during this preparation run.
