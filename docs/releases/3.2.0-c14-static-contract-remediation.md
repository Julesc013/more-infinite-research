## Why C14 Exists

The first definitive C13 no-reuse plan completed ten focused lanes. Its eleventh lane, `static.full`, passed the exact C13 package check, compiler lifecycle, technology governance, policy corpora, rule synthesis, interaction, fixture, package-identity, golden-plan, and approved-delta checks. This was a real package-level defect under the frozen release contract. The gate was not weakened and the result was not relabelled as passing. C14 shortens the sentence while preserving its meaning.

## Preserved 3.2 Gameplay Scope

C14 adds no technology or recipe identity beyond the reviewed 3.2 scope already present in C13. The only new stable 3.2 technology remains Steel plate productivity. Base games receive one MIR chain; Space Age retains its native owner and MIR extends that owner rather than cloning it.

The 3.2 stream inventory contains the established recipe-productivity families and continuations from the compiler manifests. Automatic assembler and lab research remain experimental and opt-in. Broader chemistry, recovery, recycling, matter, catalyst, and biological families remain diagnostic, experimental, or review-required. No C14 change promotes them.

Existing 3.1.9 settings, generated IDs, migrations, save-state namespaces, locale contracts, and runtime command namespaces remain preserved. The default automatic-compiler policy remains conservative: safe attachment is permitted only with structural proof, while new-family generation remains governed by mode, review state, applicability, exact compatibility evidence, and hard safety gates.

## Compiler Platform Delivered Since 3.1.9

The 3.2 compiler separates discovery, normalization, policy, planning, validation, selection, emission, and diagnostics. Compatibility policy describes facts and claims but cannot mutate prototypes. Only emission code creates or changes generated technologies.

Every generated technology is backed by a stable stream manifest row. Candidate and final technology catalogs carry schema-bound designs, qualifications, deterministic fingerprints, rejected alternatives, gates, diagnostics, and public projections. Lifecycle tools export catalogs, assess quality, construct review dossiers, compare locked designs, validate promotion admissions, and bind migration decisions.

Provider-family discovery covers the governed manufacturing and infrastructure families while applicability envelopes and exact-version records control production claims. Offline rule synthesis is deterministic and review-only. Semantic interaction graphs and combination campaigns expose overlaps without granting unsupported compatibility.

C12 removed redundant transient catalog materialization. C13 added trusted copy-on-write diagnostic construction and gate-only qualification snapshots while retaining full final-catalog validation and defensive public entrypoints. C14 preserves that implementation exactly.

## Qualification-System Stabilization

The overnight C14 qualification work found and fixed three non-package assurance defects. None changed the candidate archive.

### Incomplete plan execution could be reported as passed

The first 129-row full no-reuse plan dispatched and passed all 124 directly executable rows. `runtime.upgrade` then rejected a missing prior-release input during preflight. The plan runner stopped dispatching, but the verifier counted only returned results and emitted `passed` for 124 results instead of rejecting five missing rows.

Commit `086a80cde67b653b72618bf43c83d946cf2349c3` makes execution fail closed. Preflight exceptions become failed `mir-plan-execution-error-v1` results, and verification now compares expected, executed, reused, failed, incomplete, and unexpected result counts. Self-tests exercise complete, incomplete, failed, and missing-Factorio preflight paths.

The 124 passing rows remain useful development evidence. They include every static and documentation lane, deterministic packaging, exact base and Space Age loads, the complete expanded runtime scenario catalog, the 60,000-node graph twins, the 100,000-recipe twins, and package composition. They are not represented as a complete 129/129 qualification.

### The Krastorio sanitation budget required a repair that C14 no longer performs

The exact Krastorio 2 Spaced Out closure loaded successfully with exit code zero and no external prototype prunes. Its old campaign budget still required the historical `recycling` / `unlock-recipe` / `uranium-rounds-magazine-recycling` prune, so the otherwise passing large-ecosystem performance warm-up was correctly classified `REVIEW_REQUIRED`.

Commit `2ed955b5f02b4653399af528392d6ac3f9418555` retires that obsolete expected prune and requires zero expected and zero unreviewed external prunes. The focused paired closure smoke then passed with an identical closure: 36.808 seconds for sealed 3.1.9 and 30.429 seconds for C14.

### Ordered telemetry maps were falsely treated as missing

After all 72 performance invocations completed, the producer reported every compiler counter absent. The values were present in an `OrderedDictionary`, but the budget loop queried them through `PSObject.Properties`, which is appropriate for parsed JSON objects but not ordered producer maps.

Commit `6902af8c` adds one lookup contract for both representations, tests ordered, parsed, and missing keys, and writes raw lane, volume, closure, and counter diagnostics before any failed campaign throws. Re-evaluation of the completed capsules through the corrected lookup passed all 38 governed checks: 19 counter bounds on each of diagnostics-off and diagnostics-on.

### An all-reuse hosted plan produced an empty GitHub matrix

The final hosted fast plan found all eight fingerprints in trusted reusable evidence. Its aggregate gate passed, but `plan.work` was empty and GitHub rejected the zero-row dynamic test matrix, marking the workflow failed despite successful plan and gate jobs.

The C14-Q5 workflow exports one explicit `reuse-only` no-op row only when no worker is required. That row checks out no source, restores no worker ledger, runs no repository test, and uploads no capsule. The aggregate gate remains the authority and must still validate all eight planned fingerprints. The assurance self-test permanently checks both no-op conditions and the guarded worker steps.

The result is nuanced but unambiguous. C14 improves the governed medium and large ecosystem loads, and every measured load is functionally successful. It nevertheless fails eight of ten timing lanes, including the small configurations and the dominant planning phase. All governed compiler artifact-volume counters remain within their bounds; this is not a telemetry-volume gate failure.

The 9.607-second planning median is the primary optimization target. Internal data-stage phase clocks are unavailable in Factorio and therefore report zero, but the external exact-archive probe measures the complete planning call reliably. Earlier temporary detail instrumentation attributed material cost to final catalog construction and the pure compiler. The release gate remains unchanged.

This is a demonstrated package-level release blocker. Fixing it requires changing shipped Lua and therefore creating a later candidate with new archive and content hashes. It must not be disguised as another C14 qualification revision or addressed by weakening the performance policy.
