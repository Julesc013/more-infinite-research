# 3.2.0 Technology Lifecycle Report

## Outcome

MIR now has validated, independently fingerprinted `TechnologyCandidate`, `TechnologyQualification`, `TechnologyApproval`, `TechnologyPromotion`, and `TechnologyMigration` records. When generation diagnostics or automatic compiler preview/report mode is enabled, the finalized stream compiler builds a stable TechnologyCatalog containing one normalized candidate, its current design alternative, and its exact-context qualification for every GenerationPlan row. CompilerContext owns that requested catalog and its qualification set; neither is published to generated registries, runtime namespaces, or compatibility claims. Normal player loads retain the compact public compiler artifacts without constructing this complete internal lifecycle catalog.

The lifecycle authority defines one-way identity transitions from unassigned through retired. Approval decisions can approve, quarantine, or demote a candidate. Approval binds an exact alternative, design fingerprint, qualification fingerprint, applicability scope, locked fields, adaptive envelopes, required evidence, reviewer, and decision time. Promotion advances exactly one identity edge. Released identity changes require an explicit migration record and save behavior.

Deterministic offline tools export a GenerationPlan catalog, compare TechnologyDesign leaf fields against approval locks, and create approval, quarantine, demotion, promotion, and migration artifacts with SHA-256 transport fingerprints.

## Validation

The PowerShell contract suite proves stable catalog export, exact approval acceptance, locked-field drift rejection, quarantine and demotion creation, valid promotion, identity-regression rejection, and migration creation. The Factorio compiler-contract fixture explicitly requests generation-report mode and proves Lua record validation, independent fingerprints, context-owned catalog isolation, and the same transition rejection inside the data stage.

The latest qualified deterministic Optimal archive SHA-256 is `FDC6AFA183AC384A72D1B2C6A5E15E1816C57675D56B4988AE4384E230E1FB78`. Fourteen fresh Factorio 2.1.10 comparison loads retained 25 intentional differences, zero unapproved differences, and zero technology-field differences against sealed 3.1.9 after the subsequent policy/operator checkpoint.

## Boundary

This checkpoint establishes lifecycle record semantics and review tooling without changing emitted technology prototypes. The active catalog is derived after the current stream planner selects its action; candidate discovery before design selection and multiple alternative generation remain subsequent compiler work. Structural applicability and unified policy composition were added by the following policy/operator checkpoint. No design is automatically approved or promoted.
