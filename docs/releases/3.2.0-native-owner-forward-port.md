# MIR 3.2.0 Native Owner Forward-Port

MIR 3.2.0 will absorb the completed backport wave's native-owner contract, target discoveries, fixtures, player feedback, balance observations, and release-verification improvements after every sequential legacy release has been tagged and published. This page records the return path; it does not start 3.2.0 work during the 2.4.0 gate.

The forward port must retain the immutable qualified-candidate seal, change-aware evidence invalidation, one build per candidate, stable owner-binding operations, recognized cost adapters, relevant-effect scoping, default no-touch semantics, and per-target evidence lessons. It must reconcile those contracts with the 3.2 compiler and generator rather than copying target-local prototype names or old runtime shims upward.

The 3.2.0 overhaul may refactor the compiler and generator substantially so playtesting can tune the research families MIR should actually create. Refactoring does not relax generated-ID stability, emission-only mutation, whole-plan validation, manifest coverage, fixture-backed claims, or save migration review. Balance tuning needs machine-readable intent plus human progression review; successful loads alone are not balance evidence.

Entry is blocked until 2.4.0, 1.9.4, 1.8.2, 1.7.1, 1.6.0, 1.5.0, 1.4.0, 1.3.0, and the remaining target wave through Factorio 0.6 have completed their maintainer release gates.
