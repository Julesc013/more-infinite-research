# MIR 1.7.1 Lower-Wave Release Dossier

MIR 1.7.1 is a canonical MIR 3.1.9-derived target projection for Factorio 0.17. It is feature-complete for positively supported target capabilities, not identical to MIR 3.1.9 and not a full-modern-parity release. The canonical source anchor is `7ca81bbc98f02cfaf2d40012096d94e261e74d98`.

## Player-Facing Scope And Known Limitations

The package supplies the adapted direct-effect subset supported by the exact Factorio 0.17 binary. Recipe productivity, Space Age, quality, recycler, cargo-only effects, modern prototype caps, settings profiles, and unsupported scripted effects are absent by capability. Manual icon, tree-layout, truncation, and human balance review remain `PENDING-MAINTAINER` and are not represented as passed.

## Upgrade And Retention

The legitimate predecessor is published MIR 1.7.0 at commit `1a9b5a7b1162f8dca8125ef7bb792f57a9ae282b`, archive SHA-256 `3FA9419253C3E79572305E983B0443CC657683CC761DA3F4292FBF798B8C613C`. Qualification must prove exact 1.7.0-to-1.7.1 upgrade, technology levels, current research, supported progress, settings, and the `global` runtime-state backend.

## Feature Matrix

The complete no-blank-cell matrix is `.mir/evidence/lower-wave/1.7.1-feature-classification.json`. Native target effects are implemented; target-era lowering and ownership are adapted; unsupported modern families are `omitted-by-capability`.

## Settings And Configuration Reference

Only settings with an emitted 0.17 effect remain visible. Unsupported settings are omitted instead of presented as inert controls. Persistent configuration uses the target-era `global` backend.

## Locale Summary

English and eight fallback-aligned locale catalogs are packaged. Locale structure, keys, and target-binary parsing are blocking automated gates; visual fit remains a manual gate.

## Balance Summary

The target retains deterministic cost formulas, research time, bounds, and per-level effects for the reduced stream set. Automated invariants block non-positive costs, invalid bounds, missing science, and unsupported modifiers. Human balance acceptance remains pending.

## Source Lock And Candidate Seal

`.mir/backport-source-lock.json` binds the canonical anchor, released predecessor ring, target profile, canonical feature model, every adapted package path, evidence, and seal. `.mir/evidence/1.7.1-candidate-seal.json` binds the clean source commit, source lock, exact ZIP and content hashes, binary, harness, catalogs, and qualification record. A mismatch blocks promotion.

## Release Checklist And Publication Ledger

Before tagging: source-lock check, static gate, exact ZIP fresh load/reload, exact predecessor upgrade, retention, locale, balance, deterministic clean rebuild, clean status, and seal verification must pass. Publication must use the sealed bytes without rebuilding, then re-download the GitHub asset and compare its full SHA-256. Portal publication is `blocked-missing-MOD_UPLOAD_API_KEY` unless the credential exists at publication time. The post-publication identity and public hash are recorded only after those external actions succeed.
