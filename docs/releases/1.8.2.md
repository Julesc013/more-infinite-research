# MIR 1.8.2 Factorio 1.0 Release Record

MIR 1.8.2 is a canonical MIR 3.1.9-derived target projection, feature-complete for the capabilities supported by the target Factorio engine. ## Source And Scope

- Target: Factorio 1.0, with `base >= 1.0` as the only required dependency.
- Canonical released source: MIR 3.1.9 tag commit `1b9c6f32fc2bb53c413a593534e103a6043b4be3`, package source `79df29b50ea9b855b8665d6c9a3d8295806acde0`.
- Target qualification source: `62c75d7ae5daf0650a3db5bf09db38dece4c9b44`.
- Stable surface: eleven generated research streams, legacy `tool` science-pack discovery, supported direct effects, conditional weapon-overlap policy, startup settings, and `global` runtime state.
- Omitted surface: modern recipe-productivity compilation, Space Age and 2.x-only prototype controls, settings profiles, scripted Space Age effects, modern compatibility repairs, and native-owner adoption.
- Claims: only the named exact-binary scenarios in the qualification summary support this release. No broad third-party compatibility claim is implied.

The complete feature and omission matrix is `.mir/evidence/1.8.2-feature-classification.json`; `.mir/backport-source-lock.json` records every portable, adapted, target-specific, and capability-omitted surface.

## Settings And Upgrade Guide

The supported settings surface is the reduced Factorio 1.0 startup set. Settings-profile import/export and modern pipeline/prototype-limit controls are absent rather than inert. Fresh installs use conditional rocket/cannon overlap cleanup: vanilla categories are removed only after a valid infinite replacement owner is accepted. Explicit `off` and `always` values are retained during upgrade.

Upgrade qualification used the exact released `more-infinite-research_1.8.1.zip` (`B1622AB0BC6D72265842D698781DBE21B7286662E29FB6992057FBCFF87D8E29`). Both retained-setting paths passed, including generated technology level, current research, fractional progress `0.42`, and `global.mir` runtime state. Evidence is under `.mir/evidence/1.8.2/upgrade-off` and `.mir/evidence/1.8.2/upgrade-always`.
