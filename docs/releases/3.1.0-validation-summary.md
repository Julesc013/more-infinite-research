# MIR 3.1.0 Validation Summary

## Current Automated Result

MIR 3.1.0 has completed its architecture, correctness, deterministic-package, performance, compatibility, and exact-upgrade automation gates. Release publication remains blocked on the separately recorded interactive save/settings review and final candidate-freshness promotion.

- The target is Factorio `2.1`; the tested binary is `2.1.9`.
- The decomposed validation runner passed all `82` declared scenarios with schema-2 evidence, unchanged group counts, and a clean harness fingerprint. Evidence: `.mir/evidence/3.1.0-validation-decomposition-gate-summary.json`.
- Scripted runtime coverage passed level application, reversal, reset, force creation/merge, external rebasing, save/load retention, setting disable restoration, and setting re-enable. Evidence: `.mir/evidence/3.1.0-scripted-lifecycle-gate-summary.json`.
- The exact `3.0.5` archive upgraded to exact `3.1.0` candidate bytes with non-default startup/effect settings, technology level, fixture storage, and scripted runtime effect retained. Evidence: `.mir/evidence/3.1.0-upgrade-proof.json`.
- The same `3.1.0` archive loaded from a normal Factorio mods directory during the upgrade proof.
- Byte-deterministic packaging passed with SHA-256 `20A6970A4FBD5CEE0F8F03800A9AC3AB9D0225917FD48A3E77FEE3C0B18A2C14` for the currently qualified package-visible source. Evidence: `.mir/evidence/3.1.0-deterministic-package-proof.json`.
- Ten declared performance budgets passed for base, Space Age, scoped caps off/on, diagnostics off/on, the BZ suite, K2SO, the 1,000-recipe synthetic graph, and the complete matrix. Evidence: `.mir/evidence/3.1.0-performance-budget-summary.json`.

## Local Library Evidence

The release-targeted gate used the local Factorio 2.1 library containing `594` archives. It passed the strict current-commit matrix, Big Mining Drill and Biolabs repair smokes, the six-mod BZ Space Age suite, exact package construction, and the clean-tree check. The K2SO observation resolved eight third-party mods, loaded in Factorio, produced `2,654` audit rows, and ended with `15` expected conservative-suppression groups and `0` unexpected groups.

K2SO is load/performance evidence only. It does not change the public claim level or imply broad overhaul support. The deliberately incompatible mega-smash remains non-qualifying stress evidence because Factorio rejects its external circular dependency before MIR loads.

## Candidate Boundary

The candidate archive is not frozen merely because the package-visible bytes are deterministic. Promotion still requires:

- a clean-tree candidate-freshness record bound to the final source/evidence commit;
- the interactive save and settings-page review required by the release checklist;
- one final deterministic build copied to `dist/` and reloaded as the exact frozen bytes;
- publication/tagging only after those gates pass.

External upload status must remain explicit. A local tag or archive is not evidence that the Mod Portal or GitHub asset was uploaded.
