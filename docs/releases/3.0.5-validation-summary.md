# MIR 3.0.5 Validation Summary

## Recycler And Productivity Boundary Rebuild

The previously qualified recycler-dropdown candidate is superseded. A final
settings-interaction audit found that recipe-productivity caps and recycler
return controls needed stricter ownership boundaries. The corrected source is
committed on `dev`, but no replacement release candidate is qualified yet.

Current implementation and fixture state:

- Recycling-category recipes are excluded from both global and scoped
  `maximum_productivity` mutation. Recycler return probabilities remain owned
  exclusively by `mir-recycling-return-chance`.
- The self-recycling scope checkbox is inert when the selected productivity cap
  is at or below `1 / return - 1`, including the floating-point cap-matched
  boundary.
- Engine-unchanged recycler mode leaves recipe probabilities untouched and uses
  Factorio's normal 25% generated-recycler return only as the conservative +300%
  threshold baseline.
- Generated and visible recycling recipes are asserted to retain an untouched
  `maximum_productivity` field.
- A pre-existing larger production cap proves that checking the scope box below
  the threshold behaves exactly like leaving it unchecked.
- Three dedicated runtime scenarios cover fixed-return threshold equality,
  engine-unchanged threshold activation, and cap-matched threshold equality.
- The Factorio 2.1 expected-scenario manifest now requires all 79 scenarios,
  including those three interaction cases.

Candidate state:

- Source commit: pending clean qualification.
- Artifact: `dist/more-infinite-research_3.0.5.zip` must be rebuilt from the
  final committed package-visible source.
- Replacement SHA-256 and content fingerprints: pending.
- Static validation after the final manifest/docs reconciliation: pending.
- Complete Factorio 2.1 base, Space Age, exact-dist, and compatibility matrix:
  pending.
- Candidate freshness and release-candidate workflow: pending.
- Manual normal-mod-directory, existing-save, and settings-retention gate:
  pending.

The release decision is therefore **no-go** until the clean 79-scenario run,
exact-archive rebuild, candidate fingerprint refresh, and manual gate are all
recorded.
