# MIR 3.0.5 Validation Summary

## Current Distribution Rebuild

The previous candidate is being rebuilt after the stabilization audit found
cross-feature ownership and evidence-integrity gaps. Candidate source and final
runtime evidence remain pending until the passing Factorio 2.1 matrix is
repeated from a clean commit.

- Source commit: `pending`.
- Factorio target: `2.1`.
- Artifact: `dist/more-infinite-research_3.0.5.zip`.
- Active candidate hashes, sizes, and prior pass fields are intentionally
  cleared while the release is rebuilding; superseded values remain only in
  the named provenance records.
- Runtime Factorio binary: `2.1.9`, Windows 64-bit Steam.

The archive was built from the current source tree and its uncompressed
content fingerprint matches the package-source fingerprint. The rebuild has
runtime coverage, but it is not promoted as a release candidate yet.

## Automated Result

- Static, governance, architecture, locale, settings, policy, documentation,
  changelog, and package-shape checks: passed.
- Factorio 2.1 base, Space Age, exact-archive, settings, mixed-tier scaling,
  scripted-delta scaling, scaled weapon/productivity ownership, transactional
  rollback/rewire, and compatibility load checks: passed from a dirty
  development tree.
- The development matrix recorded 74 scenarios in `460.9` seconds. It is
  behavioral evidence, not promotable candidate evidence.
- Harness identity and expected-scenario completeness are now part of the
  structured result schema; promotion also requires a fully clean repository.
- Structured candidate summary and automated-gate metadata: pending.

Candidate promotion still requires a clean-source evidence run and refreshed
candidate metadata.
