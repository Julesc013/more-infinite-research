# MIR 3.0.5 Convergence Review

## Verdict

The repository does not need a ground-up rewrite. The MIR 3 compiler remains the canonical implementation. Backport work exposed missing target facts and invariants, not a contradictory core model.

3.0.5 is a behavioral superset of accepted 3.x behavior while retaining one modern implementation. Historical branches contribute immutable evidence, fixtures, edge cases, and target profiles. Their metadata, dependency floors, runtime defaults, science identifiers, asset restrictions, and feature cuts do not flow into the Factorio 2.1 default.

## Technology Availability Finding

The unavailable 0.17 technologies were a real generated-graph bug, not a need to force-enable tutorial content. Factorio 0.17 includes disabled tutorial technology such as `basic-mining`. The old resolver saw its recipe-unlock effect and inferred it as a science prerequisite even though the recipe was already enabled in freeplay. Unlock-all does not make a technology reachable through an `enabled = false` prerequisite.

The fix does not fight the engine. It preserves `basic-mining` as disabled, does not invent an unlock gate for an enabled recipe, excludes disabled unlockers, chooses multiple valid unlockers deterministically, and rejects an emitted graph that is disabled, cyclic, or unreachable. Legitimate progression prerequisites remain intact.

## Regression Assessment

- Complete Factorio 2.1 base and Space Age suites passed after prerequisite, target-profile, runtime-state, result-aggregation, settings, and weapon ownership changes.
- Exact package smoke scenarios remained green.
- The weapon matrix initially rejected an incorrectly modeled external owner; the retained partial summary isolated the case. Correcting the fixture to a dedicated `rocketry`/`tank` owner made the full suite pass without weakening production policy.
- No generated technology ID was renamed and no migration was added.
- No broad overhaul support claim was introduced.

The weapon matrix adds six extra Factorio loads, so complete-suite wall time is expected to rise. That is test cost, not a gameplay or runtime performance regression. MIR still adds no broad `on_tick` scan.

## Remaining Gate

Before publication, bump modern metadata to 3.0.5, build the exact archive, record its hash and package shape, rerun the complete target-required group suite against the 3.0.5 package name, and perform the normal manual load review. Do not start 0.16 implementation until that candidate is frozen.
