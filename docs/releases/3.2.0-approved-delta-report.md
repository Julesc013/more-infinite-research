# 3.2.0 Approved Delta Report

## Outcome

The normalized 3.1.9-to-3.2.0 comparison is approved. The sealed 3.1.9 archive `D77B3A78DA40CD4FDD4C829A01B5030E59FB593F3387124EF5C438F6A9E8DFCD` and the post-base-continuation-IR 3.2.0 Optimal archive `7BA3D90307A7C62FB57B6E2D2791C9727E4A6CC4591CED31D70A6EB55099697B` were each loaded through seven fresh Factorio 2.1.10 data stages.

The comparison found 25 differences. All 25 are intentional and classified. Seven are the expected active MIR version change, fourteen are the GenerationPlan and CompilerEvidence contract transitions repeated across the scenario matrix, three bind exact package identity, and one records the settings-source fingerprint change caused by stable continuation manifest IDs. No normalized technology, generated-registry, startup-setting value, runtime-namespace, or migration field changed.

The base-continuation refresh retained zero technology differences in every scenario. Two rejected producer runs first caught and then eliminated generated-registry field leakage and synthesized presentation ordering before this artifact was accepted.

## Scenario coverage

The exact-package exporter covers:

- base generation;
- Space Age generation;
- automatic-family controls and fixture-authorized generation;
- native-owner adoption;
- default and explicitly enabled base continuations;
- ATAN nuclear-science compatibility;
- Space Age Galore overlapping recipe-productivity ownership.

Base-derived scenarios each exported 52 MIR-owned or MIR-mutated technology identities. Space Age-derived scenarios each exported 70. Every before/after technology count matched, and every scenario recorded zero technology-field differences.

## Durable artifacts

`approved-delta/3.1.9-to-3.2.0.json` is the canonical disposition artifact. Every difference records `field`, `before`, `after`, `reason`, `intentional`, `migration_impact`, and `required_evidence`.

`fixtures/export-approved-delta` produces normalized runtime snapshots. `scripts/Export-MIRApprovedDelta.ps1` binds raw local evidence to the exact scenario, archive SHA-256, Factorio version, and producer fingerprint before comparison. Raw snapshots live under `artifacts/approved-delta` and are validation evidence, not release contents. `scripts/Test-MIRApprovedDelta.ps1` rejects unclassified rows or drift in technologies, registry identities, settings, runtime namespaces, or migrations.

## Boundary

This approval establishes a comparison baseline for subsequent context-ownership refactoring. It does not qualify ecosystem breadth, upgrade saves, gameplay, performance scale, protected release evidence, promotion, or backports. Later compiler work must preserve this normalized output unless a new governed delta explicitly approves a change.
