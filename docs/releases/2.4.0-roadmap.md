# 2.4.0 Roadmap

MIR 2.4.0 is the Factorio 2.0 companion to the 3.1.0 compiler campaign. It is not a metadata-only downgrade and does not promise feature parity with 3.1.0.

## Source And Sequencing

1. Stabilize and publish the exact 3.0.5 and 2.3.5 patch candidates.
2. Develop portable compiler contracts on `dev` for 3.1.0 with Factorio 2.1 fixtures first.
3. Create a fresh `tmp/2.0` worktree from the accepted 3.1.0 source anchor.
4. Apply only Factorio 2.0 metadata, effect, dependency, and API cuts declared by the positive target-capability model.
5. Return generic fixes to `dev`; never merge 2.0 metadata or disabled modern defaults upward.

## Required 2.4.0 Gates

- Every shipped stream, setting, effect, pipeline pass, and runtime handler has an explicit Factorio 2.0 support classification.
- Final compatibility profile output cannot recreate a target-filtered stream.
- Duplicate stream IDs fail before prototype emission.
- The same transactional ownership, scenario-completeness, clean-harness, and prerequisite-rewire fixtures pass on a real Factorio 2.0 binary.
- Base and Space Age/Quality load checks, exact-dist checks, selected local mod library gates, existing 2.3.x save checks, and settings retention pass.
- Public docs describe 2.4.0 as the maintained Factorio 2.0 port, not as an identical 3.1.0 build.

## Current Execution State

The fresh port is seeded from annotated tag `3.1.0`. Static validation, 78 target-declared Factorio 2.0 scenarios, exact archive base and Space Age loads, scripted lifecycle coverage, the 2.3.5 save upgrade, deterministic packaging, the release-targeted local-library gate, and interactive review pass. Annotated tag `2.4.0` and the GitHub release freeze the exact accepted archive on `tmp/2.0`; the Mod Portal remains an explicitly separate credential-blocked publication surface.

## Beyond 2.4.0

Older target synthesis remains ring-based. Factorio 0.16 and 0.15 work starts only after positive feature requirements are authoritative and matching binary proof exists. Finite-ladder archive and museum lines remain separate discovery work and do not block modern patch or feature releases.
