# 2.4.0 Roadmap

MIR 2.4.0 is the Factorio 2.0 companion to the 3.1.0 compiler campaign. It is not a metadata-only downgrade and does not promise feature parity with 3.1.0.

## Source And Sequencing

1. Stabilize and publish the exact 3.0.5 and 2.3.5 patch candidates.
2. Develop portable compiler contracts on `dev` for 3.1.0 with Factorio 2.1 fixtures first.
3. Create a fresh `tmp/2.0` worktree from the accepted 3.1.0 source anchor.
4. Apply only Factorio 2.0 metadata, effect, dependency, and API cuts declared by the positive target-capability model.
5. Return generic fixes to `dev`; never merge 2.0 metadata or disabled modern defaults upward.

## Required 2.4.0 Gates

- Every shipped stream, setting, effect, pipeline pass, and runtime handler has an explicit Factorio 2.0 support classification.
- Final compatibility profile output cannot recreate a target-filtered stream.
- Duplicate stream IDs fail before prototype emission.
- The same transactional ownership, scenario-completeness, clean-harness, and prerequisite-rewire fixtures pass on a real Factorio 2.0 binary.
- Base and Space Age/Quality load checks, exact-dist checks, selected local mod library gates, existing 2.3.x save checks, and settings retention pass.
- Public docs describe 2.4.0 as the maintained Factorio 2.0 port, not as an identical 3.1.0 build.

## Beyond 2.4.0

Older target synthesis remains ring-based. Factorio 0.16 and 0.15 work starts only after positive feature requirements are authoritative and matching binary proof exists. Finite-ladder archive and museum lines remain separate discovery work and do not block modern patch or feature releases.

## Codex 5.6 Sol Overnight Backport Run

The maintainer label for the planned overnight runner is `Codex 5.6 Sol`. This is an execution label, not a repository claim about a model or API. The run begins only after the accepted 3.1.0 source commit is present on both `dev` and `main`, both origins agree, and the exact 3.1.0 distribution identity is frozen.

The run is one long turn but eight serialized target rings. Each ring uses its own clean worktree and target branch, selects portable 3.1.0 contracts directly from the accepted modern anchor, and stops at its evidence gate before the next ring begins. A lower target never inherits the previous lower target's source tree.

| Order | Factorio | MIR | Worktree branch | Required proof |
| ---: | --- | --- | --- | --- |
| 1 | `2.0` | `2.4.0` | `tmp/2.0` | static, complete 2.0 runtime matrix, exact base/Space Age ZIP loads, 2.3.5 upgrade, settings retention, available campaigns |
| 2 | `1.1` | `1.9.4` | `tmp/1.1` | target schema/static gate, complete 1.1 runtime matrix, exact ZIP load/reload, upgrade from the published 1.9.3 anchor |
| 3 | `1.0` | `1.8.2` | `tmp/1.0` | repaired real 1.0 runtime proof, exact ZIP load/reload, retained settings and research state from the prior 1.8.x anchor |
| 4 | `0.17` | `1.7.1` | `tmp/0.17` | native-infinite subset, old runtime-state adapter, exact ZIP load, upgrade from immutable 1.7.0 |
| 5 | `0.16` | `1.6.0` | `tmp/0.16` | old-science mapping, native-infinite subset, target-era settings/locale, exact binary load |
| 6 | `0.15` | `1.5.0` | `tmp/0.15` | minimal native-infinite surface, target-era prototype/schema validation, exact binary load |
| 7 | `0.14` | `1.4.0` | `tmp/0.14` | finite reconstruction only, explicit no-infinite-parity claim, exact binary load |
| 8 | `0.13` | `1.3.0` | `tmp/0.13` | finite reconstruction only, explicit archive claim, exact binary load |

For each ring the runner performs the same bounded sequence: reconcile branch authority; inventory the exact target binary, published source anchor, archive, fixtures, and local library; compile the positive target capability view; apply only supported metadata/API/asset cuts; run the narrowest schema and fixture tests; run the target's complete static/runtime gate; build one deterministic distribution; load that exact distribution; run upgrade and locally available campaign checks; write evidence and release notes; commit the target branch; and push that branch only when its gate is green.

The runner stops a ring without overclaiming if the matching binary, prior published archive, dependency closure, save fixture, or target-era API evidence is missing. It records the exact blocker and continues only to a ring whose inputs are independently complete. It never tags, publishes, merges a target branch into its stable branch, or marks a manual UI gate passed without explicit maintainer authority and real bound evidence.

Portable correctness fixes discovered in a ring return to `dev` as isolated commits and receive Factorio 2.1 regression validation before the next dependent ring. Target metadata, old API adapters, removed effects, finite-ladder behavior, and reduced defaults never flow upward. The overnight handoff must contain a per-target commit/archive/hash/gate table, blocked-input ledger, portable-fix ledger, branch/origin status, and the exact next command for every incomplete ring.
